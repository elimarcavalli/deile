## Issue #531 — DEILE deve subir sem o bot

### PASSO 1 — Reprodução dos pontos de falha (AC7)

**Ponto de falha 1 — `Dockerfile:40-42,56`:**
`COPY deilebot/...` e `pip install ./deilebot[discord,client]` eram incondicionais.
Qualquer `docker build` sem a pasta `deilebot/` no contexto falhava no COPY antes de
chegar ao `pip install`. Confirmado via leitura direta do Dockerfile original.

**Ponto de falha 2 — `deploy.py:507-528` (`_k8s_up_resolve_profile`):**
Função retornava apenas `DeploymentProfile` sem distinguir "full explícito" de "full default".
Ambos os ramos (CLI flag inválida → fallthrough, env var ausente → fallthrough) retornavam
o mesmo objeto sem sinalizar a origem. Confirmado: `return DeploymentProfile("full")` na
linha 528 era o único retorno no caminho default.

**Ponto de falha 3 — `deploy.py:564-570` (gate de token):**
`if profile.requires_discord and not discord_token: ... return 1` era acionado
incondicionalmente para QUALQUER perfil `full`, independente de ser implícito ou explícito.
`k8s up` sem `--profile` e sem `DEILE_BOT_DISCORD_TOKEN` falhava com exit 1.

### Implementação V1

1. **`Dockerfile`**: substituído bloco `COPY deilebot/` fixo por
   `ARG WITH_BOT=0` + `RUN --mount=type=bind,source=deilebot,target=/tmp/deilebot-ctx,required=false`.
   Com `WITH_BOT=0` (default): nem o COPY nem o `pip install ./deilebot[...]` são executados.
   Com `WITH_BOT=1`: monta o contexto via bind e instala `deilebot[discord,client]`.

2. **`infra/k8s/deploy.py`**: `_k8s_up_resolve_profile` passa a retornar tupla
   `(DeploymentProfile, explicit: bool)`. `explicit=False` somente no default implícito.
   Em `k8s_up`, gate de token agora distingue: implicit full + no token → fallback `claude-only`
   + warn (AC4); explicit full + no token → erro mantido (AC6).

3. **`.env.example`**: documentada `DEILE_K8S_DEPLOY_PROFILE` com valores válidos,
   default implícito e comportamento de fallback de D2.

4. **Testes**:
   - `deile/tests/infra/test_deploy_profile_531.py` — 15 testes (D1/D2/D3/AC4/AC6).
   - `deile/tests/test_tools_import_without_deilebot.py` — 4 testes (AC3).
   - `deile/tests/infra/test_k8s_up_refactor.py` — atualizado para novo retorno tupla (6 testes).
   - Total: 25 testes adicionados/atualizados, todos verdes.
