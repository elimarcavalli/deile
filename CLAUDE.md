# DEILE — Claude Code Context

> **What DEILE is, in one breath:** a multi-provider **agent harness** (the runtime that turns an LLM API into a real tool-using agent) wrapped by an **autonomous orchestration platform**. Two faces sharing one Python core: (1) an interactive **CLI** agent, and (2) an autonomous **Kubernetes pipeline** that monitors a GitHub **or** GitLab repo, refines issues, implements on `auto/issue-N` branches, opens PRs/MRs and reviews them — with no human in the loop. It can drive **its own engine** (`deile-worker`) **or the Claude Code CLI** (`claude-worker`, `claude -p`) per pipeline stage. Harness at the core, meta-orchestrator at the edge.

This file is **loaded into every session**. It is the architecture map + how-to-work-here guide. It must contain **only verified truth**. The product-level reference is `README.md` (rewritten and code-validated, on `main`) — read it **semantically on demand** (see §1), never cover-to-cover each session.

### Operating rules
1. **The code is the source of truth.** `Comentário não é prova, o código é.` When you state how something works, cite `arquivo:linha`. No source → mark it a supposition. Never guess.
2. **Any doubt → consult `README.md` (relevant section) and/or the pillar doc, then the code.** Don't fill gaps with assumptions. Declare confidence level; separate verified from supposed.
3. **Use skills and slash commands proactively** when they raise the quality of the result — don't reinvent what a command already does. 4. **When you dispatch subagents, ALWAYS instruct each one to read `CLAUDE.md` first** (and point it at the specific `README.md` section + pillar doc relevant to its slice). Every agent must start with full architectural context — never spawn a context-blind agent on this repo. For 2+ independent, no-shared-state tasks, parallelize.
5. **Alignment before autonomous go** on non-trivial tasks: sketch the plan + risk points, get the nod. A well-specified, ready plan can proceed directly. Quality > workaround — understand impacts fully, reuse the existing architecture, refactor when needed (with prior alignment), never duplicate logic. Self-critique non-trivial work before closing ("where does this break?").
6. **Conventions** (full rules in the global `~/.claude/CLAUDE.md`): commit titles in English `type(scope): description`, body in PT-BR; **never** add `Co-Authored-By: Claude` or "Generated by Claude Code"; all user-visible docs/comments in correct **PT-BR**; prefer "Humano". `elimarcavalli` on GitHub = the Human; `deile-one` = the bot — don't assume authorship. ## 2. Mandatory protocol (run before every non-trivial turn)

Before the first `Write`, `Edit`, or mutating `Bash` of a turn:
1. Classify the **action** → consult the trigger index in `03-PRINCIPIOS-ARQUITETURAIS.md`.
2. Classify the **target path(s)** → match the subpackage map in `02-ARQUITETURA.md` / §3.1 below.
3. Match the **user's keywords** → architecture/scope/capability terms point to the relevant pillar (`00-VISAO-GERAL.md`).
4. **Union** the pillars implied; `Read` every unread one **before** mutating.
5. If scope grows mid-task, **stop and re-run** the protocol.

**Exempt:** typos, whitespace, single-line cosmetic edits, renaming a strictly-local var, read-only questions, running tests/lint/formatters, editing `.env`/lockfiles/`CLAUDE.md`/`docs/system_design/`. When uncertain → **not** exempt.

## 3. Architecture map — the real system

### 3.1 The Python package (`deile/`)
Hexagonal, registry-driven, **async-first** (no sync I/O inside `async def`). Adding a tool/command/parser/persona/skill must **not** touch the core (Open/Closed). The core never imports external SDKs directly — adapters live in `infrastructure/` or concrete providers.

| Subpackage | Responsibility | | Subpackage | Responsibility |
|---|---|---|---|---|
| `core/` | Agent loop, `ToolLoopExecutor` | | `orchestration/` | Plans, workflows, **pipeline/**, **forge/**, approval |
| `core/models/` | Providers, `ModelRouter`+`TierRouter`, streaming, `reasoning.py` | | `runtime/` | Per-process state file, Unix-socket status server, fleet registry (#303) |
| `events/` | Async `EventBus` + handlers | | `observability/` | OTLP tracer/metrics, dispatch-span adapter (#303/#48) |
| `tools/` | Tool registry + impls (+ `messaging/`) | | `cron/` | `CronStore` (SQLite) + `CronRunner` |
| `commands/` | Slash commands (`builtin/`) + registry | | `memory/` | 4 layers (working/episodic/semantic/procedural) |
| `parsers/` | Input parsing (files, diffs, refs, commands) | | `security/` | `PermissionManager`, `AuditLogger`, `SecretsScanner` |
| `personas/` | MD instructions + YAML library | | `storage/` | logger, `UsageRepository`+`BudgetGuard`, embeddings |
| `skills/` | Discovery, registry, hot-reload | | `ui/` | Rich streaming, sub-agent panel, `ui/panel/` |
| `integrations/` | HTTP client for the reverse `agent → bot` arrow | | `infrastructure/` | external adapters (SDKs) |
| `evolution/` | auto-learning (**experimental**) | | `plugins/` | plugin manager, hot-reload (**no real sandbox**) |
| `preferences/` | backing store for the `*_preference` tools | | `config/` | `Settings` singleton, YAML, profiles |
| `log_mgmt/` | log analysis/rotation/dispatch | | | |

> **Entry point:** `python3 deile.py` is a thin launcher (prepares venv) → `deile.cli.main`; the interactive shell class is **`_DeileCLI`** in `deile/cli.py` (NOT `DeileAgentCLI`). All logic lives under `deile/`.

**Inventories — verify in code, never hard-code counts:** tools = the `name` of each `Tool` subclass in `deile/tools/**` + `DEFAULT_TOOL_PACKAGES` in `discovery.py`; commands = `deile/commands/builtin/*_command.py` (`name=`/`aliases=`); models/tiers = `deile/config/model_providers.yaml`; personas = `deile/personas/{library/*.yaml,instructions/*.md}` (currently: `analyst`, `architect`, `debugger`, `developer`, `reviewer` + instr. `discord_developer`, `monitor`, `fallback`); skills = the 6 scan dirs. The current real set is inlined in §3.6–3.8 below.

**A CLI turn (the core loop):** `_DeileCLI` reads input → `DeileAgent`; parsers extract file/command refs; a slash command is dispatched via `CommandRegistry`, otherwise `ModelRouter`/`TierRouter` picks the provider/model; the provider streams `UnifiedStreamEvent`s (`TEXT_DELTA`, `TOOL_USE_START`/`TOOL_USE_END`, …); on a tool-use the `ToolLoopExecutor` runs the tool and feeds the `ToolResult` back into the conversation (cap `max_tool_iterations`, default 100, `DEILE_MAX_TOOL_ITERATIONS`); the `StreamingRenderer` paints live via `rich.live.Live`; the `EventBus` publishes telemetry/persona/tool events. Streaming-first (`process_input_stream`) is the default path.

### 3.2 The two execution engines (per-stage selectable)
- **`deile-worker`** — runs **DEILE Python in-process** (`wrapper.py worker` → `infra/k8s/worker_server.py`), uses DEILE's own provider routing.
- **`claude-worker`** — runs **`claude -p`** (Claude Code CLI) in an isolated **workdir per task_id** (`gh repo clone` into `/home/claude/work/<task_id>/repo`), OAuth creds on a PVC.

The pipeline picks the engine **per stage** via `deile/orchestration/pipeline/dispatch_resolver.py`. **5 stages × 3 axes**, each axis with a global fallback + env/settings persistence:

| Stage | Worker (`dispatch`) | Model | Reasoning |
|---|---|---|---|
| `classify` `refine` `implement` `pr_review` `follow_ups` | `DEILE_PIPELINE_DISPATCH_<STAGE>` → `DEILE_PIPELINE_DISPATCH_MODE` (default `deile-worker`) | `DEILE_PIPELINE_MODEL_<STAGE>` → `DEILE_PREFERRED_MODEL` | `DEILE_PIPELINE_REASONING_<STAGE>` → `DEILE_REASONING_EFFORT` |

Resolvers: `dispatch_resolver.py` / `model_resolver.py` / `reasoning_resolver.py`. Full diagrams: **README §Pipeline autônomo**.

### 3.3 Pipeline label state machine (the operational core)
```
🆕 ~workflow:nova → 🔍 ~workflow:em_revisao ─┬─ CLARO → ✅ ~workflow:revisada
                                             │            ├─ intent → 🧩 ~workflow:decomposta (architect opens N derived)
                                             │            └─ code   → 🚀 ~workflow:em_implementacao → 📬 ~workflow:em_pr
                                             └─ VAGO → 🏷️ refinar + (🧠 em_refinamento [intent→analyst]
                                                                      │  or 🏛️ em_arquitetura [feat/bug/refactor→architect/debugger])
                                                      ↕ ⏸️ ~workflow:aguardando_stakeholder (human removes to lift)
                                                      → back to ~nova (up to 5 rounds, durable via ~refine:N → ⛔ ~workflow:bloqueada)
```
- **Refinement gate** (persona by issue type: `intent→analyst`, `feature`/`refactor`→`architect`, `bug→debugger`); VAGO issues refined up to **5 rounds** (counter in the `~refine:N` label, survives restarts); high-impact gaps pause in `aguardando_stakeholder`. Intents that pass are **decomposed** by `architect` (anti-flood default: aggregate into ONE derived issue with a checklist). `max_parallel=2`, worker scaled to 2 replicas.
- **PR labels:** `~review:pendente → ~review:em_andamento → ~review:concluida`. Locks: `~batch:<sha8>` (**only when `shard_count>1`** — a single monitor skips it, no churn), `~by:<id>` (owner). Markers: `~mention:processado` (idempotency), `~workflow:bloqueada` (hard block, **excludes from auto-resume** — a human removes it).
- **Mention/assignment routing** — `process_mentions` is a router: issue+assignee/body → inject `~workflow:nova`; **any** PR trigger → unified `pr_unified` brief (worker reads real PR state and acts); issue+comment → one-shot under `developer` persona. Anti-eco: comments by `gh_login` are dropped. `CommentRef.is_pr_comment` routes PR comments correctly.
- **Resume vs reaper (two separate paths/counters):** *resume* continues paused work (`resume_max_attempts=10`); the *reaper* re-claims **stale** claims (`em_implementacao`/`em_andamento` past a ~45 min threshold) via `~attempt:N` with `reaper_max_attempts=3` — at attempt 3 it **blocks** (`~workflow:bloqueada`) instead of releasing. Resume itself: fresh by default; only with `resume=True` **and** a `DispatchLedger` entry; an over-budget `claude -p` session is **promoted to fresh**. Recurring `WORKER_AUTH_EXPIRED` → backoff `min(60·2^n, 1800)s` per target after 3 fails.
- **One tick (`PipelineMonitor.tick`):** **reaper FIRST**, then stages run **reconcile-before-dispatch** (completed fire-and-forget work frees capacity in the same tick, #373) and **resume before claiming new work**: reconcile → classify → review → refine → **resume** → implement → decompose → pr_review → pr_triage → mentions. A schedule file (cron windows) is authoritative if present, else every stage runs every tick.
- **Priority & triage:** `~prioridade:0..3` labels actually **reorder** candidate selection (lower first; unlabeled last; PRs inherit from their linked issue). **PR triage** labels `~review:pendente` only on branches this monitor owns (`auto/issue-*`, or `<prefix>/` when sharded), OR any PR if `enable_review_human_prs=True` (default **False**) — PRs with an empty `head_ref` (cross-repo/fork) are **silently skipped**.
- **Quality gate (READ — it is NOT what you'd assume):** the **PR/review brief** (`_WORKER_PR_BRIEF`) runs the **full suite** `python3 -m pytest deile/tests/ -q` before approve/merge. The **implementation briefs** (`implement`/`implement_resume`) run **only the impacted subset** `pytest <impacted> -p no:cov -q` and **explicitly forbid** the full suite — that's the reviewer's job (`_IMPACT_TEST_STRATEGY`, `briefs.py:59-67`). The full-suite command carries **no coverage gate** (plain `-q`; `pytest.ini` has no `--cov-fail-under`). The merge gate is **prompt-enforced by the agent**, not a deterministic CI trap. All briefs: `deile/orchestration/pipeline/briefs.py`.

**Pipeline package map** (`deile/orchestration/pipeline/`): `monitor.py` (tick loop + config), `stages.py` (per-stage logic + mention routing), `briefs.py` (**all** worker prompts), `labels.py` (the state machine), `dispatch_ledger.py` (anti-double-dispatch), `{dispatch,model,reasoning}_resolver.py` (per-stage routing), `follow_up_detector.py` + `post_merge_callback.py` (writes an episodic memory per merged PR), `identity.py` (`MonitorIdentity` + sharding + `~by:<id>` lock), `claude_dispatcher.py`, `cost_estimator.py`, `reset.py`, `worktree_manager.py`.

> **Label edits must use REST, NOT `gh issue/pr edit --add-label`** (that runs a GraphQL `login` query needing `read:org`, which the pipeline token lacks). The forge layer does REST in `deile/orchestration/forge/github_forge.py`. Manual:
> ```bash
> gh api -X POST   "repos/elimarcavalli/deile/issues/<N>/labels" -f 'labels[]=~workflow:revisada'
> gh api -X DELETE "repos/elimarcavalli/deile/issues/<N>/labels/%7Eworkflow%3Arevisada"   # ~=%7E :=%3A
> ```

### 3.4 Other subsystems — working notes
- **Reasoning effort** (`deile/core/models/reasoning.py`, single source, fail-open): vocabulary `low/medium/high/xhigh/max/ultracode/auto` (Claude-Code style), translated to each provider's native param. Global `DEILE_REASONING_EFFORT` / `/reasoning` in CLI; per-stage above. On claude-worker, `ultracode` → `xhigh` + a prompt preamble (needs `claude ≥ 2.1.158`).
- **Skills** — pure-Markdown, no Python. 6 scan dirs (precedence low→high: bundled `deile/skills/library/**` < `~/.deile/skills` < `~/.claude/commands` < `<cwd>/.deile/skills` < `<cwd>/.claude/commands` < configured paths). 4 auto-injection triggers (`file_globs`/`code_block_langs`/`keywords`(escaped literals)/`file_content_patterns`(4 KiB, project-root-contained)). Hot-reload ~0.5 s. Bundled: `python`, `typescript`, `tdd`. Also reachable via `invoke_skill`/`list_skills`/`/<name>`.
- **Sub-DEILEs (parallel decomposition)** — the `dispatch_parallel_subagents` tool splits a request into 2–5 independent sub-tasks (clean sessions, live panel). `LocalSubAgentRunner` (default, in-process) or `WorkerSubAgentRunner` (`DEILE_SUBAGENT_RUNNER=worker`, delegates to `deile-worker`). Tunables: `DEILE_SUBAGENT_MAX_PARALLEL` (3), `_BUDGET_S` (600).
- **Loop guard** (`deile/core/loop_guard.py`, wired into the tool-loop **and** all 3 non-streaming providers) aborts a runaway turn via 4 rules: hard ceiling (`loop_guard_max_calls=50`), ≥3 identical consecutive `(name,args)`, ≥3-of-5 sliding window, ≥6 no-progress (empty/error) results — emitting `AuditEventType.SUSPICIOUS_ACTIVITY`. Tunables: `DEILE_LOOP_GUARD_*` / `DEILE_MAX_TOOL_ITERATIONS` (default 100, distinct from the guard's 50-call ceiling).
- **OTLP names** (debugging reference) — spans `deile.turn`, `deile.tool.<name>`, `deile.llm.call`, root `deile.dispatch` (+ events `dispatch.received/model_resolved/progress/tool_burst/completed/failed`, child `git.commit`/`git.push`/`forge.pr_open`/`forge.pr_review`); metrics `deile.tokens.total`, `deile.cost.usd.total`, `deile.tool.duration_ms`, `deile.turn.duration_ms`, `deile.errors.total`. No secrets in spans; `session_id` is not a label.
- **Memory** — 4 layers via `MemoryManager`; episodic is SQLite (`aiosqlite`, `episodes.db`). Never store secrets/PII in any layer; always `await`; **open the layer module before calling — signatures evolve**, don't invent method names.
- **Runtime + observability** — each process publishes `~/.deile/run/<id>.json` (atomic write + heartbeat) + a Unix-socket status server; a `registry.json` (flock, GC dead PIDs) gives fleet view. OTLP traces/metrics (`DEILE_OTLP_*`, no-op when endpoint empty or `DEILE_OBSERVABILITY_DISABLED=true`); **no secrets in spans**, `session_id` not a label.

**Testing gotchas:** `pytest.ini` uses `--strict-markers` (register markers first — `unit, integration, security, orchestration, bash, ui, slow, perf, e2e, e2e_{discord,telegram,whatsapp,meta}_live, manual, llm`) and `asyncio_mode=auto` (no `@pytest.mark.asyncio` needed). **No `--cov-fail-under` exists** — the 80% gate is not enforced. The **full suite carries pre-existing failures** that pass in isolation (ordering pollution) — diff against a baseline before declaring a regression. `deile/tests/` mixes pytest tests (`test_*.py`, collected) with **standalone scripts** (`*_test.py`, `smoke_test_*.py`; run via `python deile/tests/all.py [--filter <sub>]`). Token-spending tests live in `deile/tests/might/`. **Changed a function signature?** `grep -rn` the whole test tree for every mock/`monkeypatch`/`patch`/fake of it and update each (a stale double throws `TypeError` → handler 500 → misleading red), then run the **full** suite.

## 5. Kubernetes / cluster operations

Cluster = **Rancher Desktop (k3s/containerd)**, single image **`deile-stack:local`** (`imagePullPolicy: Never`). `/app` is **baked at build time** (`COPY`, not mounted) → **code changes go live only after rebuild + restart**. `kubectl` lives at `~/.rd/bin/kubectl` (may not be on `PATH`). Dockerfile (Python 3.11 slim) installs `gh`, `glab` (SHA256-verified), `kubectl` v1.31.4, `procps`, `tini`, and the **`claude` CLI pinned to 2.1.158**.

### 5.4 claude-worker — dispatching `claude -p` in-cluster
- **Setup from scratch:** `deploy.py k8s claude-login` captures host `~/.claude/` creds → Secret `claude-credentials` → initContainer copies to PVC `/home/claude/.claude/credentials.json` mode `0600` (refresh in-pod under flock). `--switch` forces a new account; `--no-interactive` is CI-safe.
- **Panel `[d]` → `DispatchMatrixView`** — one row per stage, **6 editable columns**: Worker × Model × Timeout(s) × Retries × Cost-cap (USD/run) × Reasoning (Model is anthropic-only when Worker=`claude-worker`). Hotkeys: `[enter]` picker, `[r]` reset, `[L]` switch login, `[I]` install, `[U]` uninstall, `[s]` scale, `[c]` cleanup (preview+confirm via `kubectl exec`), `[p]` `max_parallel` (`[a]`=`auto`), `[J]` JSONL retention.
- **Cost ledger (#445):** `claude -p` transcripts couple `--resume` continuity (bulky, ephemeral) with cost auditing (tiny, permanent). `_do_cleanup` (startup hook + CronJob `claude-worker-cleanup` 03:00 UTC + `POST /v1/cleanup`) **harvests each orphan session's cost into a durable JSONL ledger BEFORE pruning the transcript** (`infra/k8s/jsonl_cost.py` = single pricing source; dedup by `session_id`). `infra/k8s/session_tokens_audit.py` reads ledger (pruned) + live JSONL (recent) with identical pricing. **NEVER prune without harvesting** (a prior incident deleted 337 transcripts; the harvest is the fail-safe).

| claude-worker env var | Default | Use |
|---|---|---|
| `DEILE_CLAUDE_MAX_BUDGET_USD` | `8` | hard cost cap → `claude --max-budget-usd` per dispatch (fights runaway PR cost) |
| `DEILE_CLAUDE_RESUME_TOKEN_BUDGET` | `100000` | over-budget `claude -p` session → promoted to fresh (**NOT** 500k) |
| `DEILE_CLAUDE_CLEANUP_RETENTION_DAYS` | `7` | workdir retention (startup + CronJob) |
| `DEILE_CLAUDE_JSONL_RETENTION_DAYS` | `30` | transcript retention (separate knob; panel `[J]`) |
| `DEILE_CLAUDE_JSONL_ORPHAN_GRACE_S` | `3600` | grace before pruning an orphan transcript (TOCTOU guard) |
| `DEILE_PIPELINE_MAX_PARALLEL` | `2` | max simultaneous dispatches (read by the **pipeline monitor**, not the worker) |

**Threat model (be precise):** ingress to `:8767` is restricted to `deile-pipeline` only. **Egress is NOT host-whitelisted at L3/L4** — the NetworkPolicy allows generic TCP 443 to anywhere (`to: []`) + DNS (k3s has no FQDN-aware CNI). The host/repo allowlist (`api.anthropic.com`, `github.com`, `gitlab.com`, per-repo) is enforced **only in `wrapper.py`** via ConfigMap `claude-worker-allowed-repos` (a `git` guard). Known gap (spec §7): prompt injection could exfiltrate creds through legitimate channels. Spec: `docs/superpowers/specs/2026-05-26-claude-worker-design.md`. *(Note: Decisão #43 still calls this an L3/L4 egress whitelist — inaccurate; the real control is application-level.)*

### 5.5 deile-monitor — the 24/7 cluster supervisor (6th pod)
Two-phase deterministic tick (PR #504): **Phase A** = mechanical sweep with **zero LLM** — 8 "vigias" (OAuth health, error pods, orphan issues, `auto/*` PR attempt N/3, awaiting-stakeholder, failed Jobs, pipeline-pod health, follow-up collection); **Phase B** invokes the `monitor` persona **only** when Phase-A candidates survive (signalled via a state file). Steady state burns **no tokens** (it used to burn ~3.5M/tick as a tool-loop — that was the #504 fix). Tick default 30 min; RBAC `deile-monitor-sa` (read pods/jobs + `pods/exec` for OAuth renew). Behaviour is persona-driven (`deile/personas/instructions/monitor.md`, hot-reload).

### 5.6 Observability gotchas — what you see is NOT always reality
1. **`ps`/`pgrep` work** now that `procps` is in the image. On an old image, `cat /proc/*/cmdline | tr '\0' ' '` works (`/proc` shows the host PIDNS).
2. **Pipeline logs look empty when idle** — the monitor emits per-tick; short tails get evicted by health probes. Use `--tail=500` / `--since=10m`. Buffering is unbuffered (`PYTHONUNBUFFERED=1` + tini), so true silence-with-RUNNING means deadlock (rare) or too-aggressive a tail.
3. **`.lease.json` mtime is NOT liveness for `claude -p`** — the wrapper rewrites `heartbeat_at` every 5 s regardless of any subprocess, and the lease `pid` is the **wrapper** PID. Every dispatch also writes `claude_pid`, and `_find_active_lease` reports `claude_running` (`True` iff that PID is alive). **Trust `claude_running` (or `pgrep -a claude`), never the lease mtime.**
4. **OAuth renew is an operational trap.** The claude-worker access token expires (~hours) and the in-pod "refresh" does **not** rotate the refresh token → the whole fleet can fall into `401 / WORKER_AUTH_EXPIRED` and look like a "stuck pipeline" with no code cause. Fix from the host: **`python3 infra/k8s/deploy.py k8s claude-renew`** (reads the macOS keychain). Check this first when dispatches mysteriously stall.

DEILE is **forge-agnostic** (#297): same pipeline/briefs/tools over GitHub (cloud + GHES) and GitLab (cloud + self-hosted). `deile/orchestration/forge/` hides the difference behind `ForgeClient` (`GitHubForge` via `gh`, `GitLabForge` via `glab` + REST v4); the pipeline never calls the CLIs directly (`pipeline/github_client.py` is a deprecation shim). |---|---|---|

## 8. Task playbooks (recipes for working autonomously)

- **Ship a code change to the cluster** → edit → `deploy.py k8s build --restart --yes` (the image is baked; **no rebuild = no effect**) → confirm with `k8s status` + `k8s logs`. For a config-only change to the pipeline pod, `kubectl set env` + `k8s restart` avoids a rebuild.
- **"Pipeline looks stuck"** → (1) **OAuth first** — `deploy.py k8s claude-renew` from the host (expired token masquerades as stuck, §5.6); (2) liveness via `claude_running` / `pgrep -a claude`, never lease mtime; (3) `k8s logs deile-pipeline --tail=500`; (4) the panel `[t]` tokens or the status server `/v1/pipeline-status`. The `/babysit-deile` command arms watchers for exactly this.
- **Work on a brief / the quality gate** → briefs in `briefs.py`; remember implement=subset, PR=full suite, no cov gate; after editing run `pytest deile/tests/orchestration/pipeline/test_briefs* -v` (the brief invariants are test-guarded).
- **Tune per-stage routing/model/reasoning** → panel `[d]` matrix, or env `DEILE_PIPELINE_{DISPATCH,MODEL,REASONING}_<STAGE>` (rollout `deile-pipeline`, no rebuild).
- **Add a tool/command/parser/persona/skill/provider** → §3.5 map + templates in `12-PADROES-CODIGO.md`; then full suite (mind ordering pollution, §4).
- **Commit & integrate** → commit **per phase**; **full green suite + self-review before every commit**; English title / PT-BR body; no `Co-Authored-By`. For a finished branch, reach for `superpowers:finishing-a-development-branch`.
- **Restart the bot** → kill old instances/pods first; two live bots double-process events.
- **Visual panel change** (`_panel*.py`) → after a visual change, **offer the Human a screenshot** (SVG via `Console.save_svg`) — don't auto-generate.

## 9. Hard-won operational lessons (don't re-pay the cost)

- **claude-worker resume context measurement bug (fixed `065e3ea`):** the gate summed `cache_read` across all rounds (11.5M in a ~70K session) against a fixed 100K ceiling → every long review restarted from scratch. Fix = peak-of-1-round + 80% of the real window. `claude -p` has **no tunable auto-compact** (only via SDK). If long sessions restart, suspect context-measurement, not the model.
- **`ultracode` on claude-worker** needs `claude ≥ 2.1.158` (pinned in the Dockerfile) and injects the keyword "workflow" + `xhigh`. Bumping the pin can hit a **Docker layer-cache** trap — force a clean build layer.
- **`infra/k8s/` packaging + destructive-code fail-safe:** the module ships in the image via explicit `COPY` + a `.dockerignore` exception. **Never prune without harvesting first**, and always test the `None` path — the cost-ledger incident (337 transcripts deleted) came from pruning before harvesting.
- **Verify before asserting:** never claim "works / it's live" without empirical proof; say what you couldn't test. (`feedback_honesty_verify`.)
- **`/proc`-first skepticism** for claude-worker liveness; run a skeptical delta-loop, not a hopeful one (`/babysit-deile` encodes this).
- More incident detail lives in persistent memory (`MEMORY.md`) — consult it for cost accounting, loop-guards history, and the refinement-gate/state-machine deep notes.

## 10. Gotchas (not in the system design)

- **At least one provider key is required at startup** (`ANTHROPIC_/OPENAI_/DEEPSEEK_/GOOGLE_API_KEY`); `bootstrap_providers()` in `deile/core/models/bootstrap.py` does conditional registration.
- **`deilebot` is a separate repo** (`elimarcavalli/deilebot`). `pip install -e ".[bot]"` (git-URL extra) for cloud/CI; clone it + `pip install -e ./deilebot` to hack on the bot. The `messaging.discord_*` tools auto-register **only** when `import deilebot` succeeds **and** `DEILE_BOT_ENDPOINT` + `DEILE_BOT_AUTH_TOKEN` are set. Run the bot from the `deile/` repo root.
- **Touching messaging tools?** Open `08-SEGURANCA.md` first — DM and role-mention tools are `SecurityLevel.DANGEROUS` and gated by `ApprovalSystem` (`deile/orchestration/approval_system.py`); that gate is non-trivial.
- **`Settings` is a singleton** — `from deile.config.settings import get_settings`, never `Settings()`.
- **Personas are MD-driven** — edit `deile/personas/instructions/*.md`; no Python change.
- **`CLAUDE.md` is versioned** because nothing in `.gitignore` ignores it (`.gitignore` lists `.claude/`, `.openclaude/*`, `.claude-worktrees/` — there is **no** `*claude*` glob nor `!CLAUDE.md` negation; don't go hunting for one).
- **Discord interactions** → use native modals/buttons/selects; check the docs before implementing.
- **Don't assume the tool registry enforces permissions or audit.** The "permission-before-privileged-action + typed audit on every tool" principle is **not** wired into the central exec path (`tool_loop_executor`) — only `bash_tool` checks `PermissionManager` (and even that allows-all when no manager is injected); other tools run ungated (Decisão #29 deferred this). The audit event that fires reliably is the **loop-guard abort** (`SUSPICIOUS_ACTIVITY`). The `secrets_scanner` IS wired (session store, worker client, sub-agent dispatch) and redacts GH/GL/AWS/bearer/private-key patterns. So: rely on the loop guard + NetworkPolicy + tool whitelist for containment, not on a per-tool permission gate that mostly isn't there.

## 12. SQL / database operations

All SQL scripts are the Human's to run. On a DB error during a task, **stop and tell the operator which script to execute** — never run migrations or schema changes yourself. DEILE's own stores are SQLite, auto-created at runtime (no versioned SQL): tasks `./.deile/db/tasks.db`, usage `~/.deile/db/usage.db`, episodic `episodes.db`, cron `data/cron.db`; the claude-worker cost ledger is JSONL at `~/.claude/cost-ledger.jsonl`.
