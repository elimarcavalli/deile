# DEILE вЂ” Claude Code Context

> **What DEILE is, in one breath:** a multi-provider **agent harness** (turns an LLM API into a real tool-using agent) wrapped by an **autonomous orchestration platform**. Two faces, one Python core: (1) interactive **CLI** agent; (2) autonomous **Kubernetes pipeline** monitoring a GitHub **or** GitLab repo вЂ” refines issues, implements on `auto/issue-N` branches, opens PRs/MRs, reviews them, no human in the loop. Drives **its own engine** (`deile-worker`) **or the Claude Code CLI** (`claude-worker`, `claude -p`) per stage.

**Loaded into every session** вЂ” the architecture map + how-to-work guide; **only verified truth**. Product reference = `README.md` (code-validated, on `main`) вЂ” read it **semantically on demand** (В§1), never whole.

4. **Dispatching subagents в†’ ALWAYS tell each to read `CLAUDE.md` first** (+ the specific `README.md` section + pillar doc for its slice). Never spawn a context-blind agent here. Parallelize 2+ independent, no-shared-state tasks.

Quality > workaround вЂ” understand impacts, reuse the architecture, refactor when needed (with prior alignment), never duplicate logic. Self-critique before closing ("where does this break?").
6. **Conventions** (full rules in global `~/.claude/CLAUDE.md`): commit titles in English `type(scope): description`, body in PT-BR; **never** add `Co-Authored-By: Claude` or "Generated by Claude Code"; all user-visible docs/comments in correct **PT-BR**; prefer "Humano". `elimarcavalli` on GitHub = the Human; `deile-one` = the bot вЂ” don't assume authorship. Before the first `Write`, `Edit`, or mutating `Bash` of a turn:

## 3. Architecture map вЂ” the real system

### 3.1 The Python package (`deile/`)
Hexagonal, registry-driven, **async-first** (no sync I/O inside `async def`). Adding a tool/command/parser/persona/skill must **not** touch the core (Open/Closed). The core never imports external SDKs вЂ” adapters live in `infrastructure/` or concrete providers.

| Subpackage | Responsibility | | Subpackage | Responsibility |
|---|---|---|---|---|
| `core/` | Agent loop, `ToolLoopExecutor` | | `orchestration/` | Plans, workflows, **pipeline/**, **forge/**, approval |
| `core/models/` | Providers, `ModelRouter`+`TierRouter`, streaming, `reasoning.py` | | `runtime/` | Per-process state file, Unix-socket status server, fleet registry (#303) |
| `events/` | Async `EventBus` + handlers | | `observability/` | OTLP tracer/metrics, dispatch-span adapter (#303/#48) |
| `tools/` | Tool registry + impls (+ `messaging/`) | | `cron/` | `CronStore` (SQLite) + `CronRunner` |
| `commands/` | Slash commands (`builtin/`) + registry | | `memory/` | 4 layers (working/episodic/semantic/procedural) |
| `parsers/` | Input parsing (files, diffs, refs, commands) | | `security/` | `PermissionManager`, `AuditLogger`, `SecretsScanner` |
| `personas/` | MD instructions + YAML library | | `storage/` | logger, `UsageRepository`+`BudgetGuard`, embeddings |
| `skills/` | Discovery, registry, hot-reload | | `ui/` | Rich streaming, sub-agent panel, `ui/panel/` |
| `integrations/` | HTTP client for the reverse `agent в†’ bot` arrow | | `infrastructure/` | external adapters (SDKs) |
| `evolution/` | auto-learning (**experimental**) | | `plugins/` | plugin manager, hot-reload (**no real sandbox**) |
| `preferences/` | backing store for the `*_preference` tools | | `config/` | `Settings` singleton, YAML, profiles |
| `log_mgmt/` | log analysis/rotation/dispatch | | | |

> **Entry point:** `python3 deile.py` is a thin launcher (prepares venv) в†’ `deile.cli.main`; the interactive shell class is **`_DeileCLI`** in `deile/cli.py` (NOT `DeileAgentCLI`). All logic lives under `deile/`.

**Inventories вЂ” verify in code, never hard-code:** tools = `Tool` subclass `name` in `deile/tools/**` + `DEFAULT_TOOL_PACKAGES` (`discovery.py`); commands = `deile/commands/builtin/*_command.py`; models/tiers = `model_providers.yaml`; personas = `deile/personas/{library/*.yaml,instructions/*.md}` (now: `analyst`, `architect`, `debugger`, `developer`, `reviewer` + instr. `discord_developer`, `monitor`, `monitor_qa`, `fallback`); skills = 6 scan dirs. Real set in В§3.6вЂ“3.8.

**A CLI turn (core loop):** `_DeileCLI` в†’ `DeileAgent`; parsers extract file/command refs; slash commands dispatch via `CommandRegistry`, else `ModelRouter`/`TierRouter` picks provider/model; the provider streams `UnifiedStreamEvent`s (`TEXT_DELTA`, `TOOL_USE_START`/`TOOL_USE_END`, вЂ¦); on a tool-use `ToolLoopExecutor` runs it and feeds the `ToolResult` back (cap `max_tool_iterations`=100, `DEILE_MAX_TOOL_ITERATIONS`); `StreamingRenderer` paints via `rich.live.Live`; `EventBus` publishes telemetry/persona/tool events. Streaming-first (`process_input_stream`) is default.

### 3.2 The two execution engines (per-stage selectable)
- **`deile-worker`** вЂ” **DEILE Python in-process** (`wrapper.py worker` в†’ `infra/k8s/worker_server.py`); DEILE's own provider routing.
- **`claude-worker`** вЂ” **`claude -p`** (Claude Code CLI) in an isolated **workdir per task_id** (`gh repo clone` в†’ `/home/claude/work/<task_id>/repo`); OAuth creds on a PVC.

Engine picked **per stage** via `dispatch_resolver.py`. **5 stages Г— 3 axes**, each with a global fallback + env/settings persistence:

| Stage | Worker (`dispatch`) | Model | Reasoning |
|---|---|---|---|
| `classify` `refine` `implement` `pr_review` `follow_ups` | `DEILE_PIPELINE_DISPATCH_<STAGE>` в†’ `DEILE_PIPELINE_DISPATCH_MODE` (default `deile-worker`) | `DEILE_PIPELINE_MODEL_<STAGE>` в†’ `DEILE_PREFERRED_MODEL` | `DEILE_PIPELINE_REASONING_<STAGE>` в†’ `DEILE_REASONING_EFFORT` |

Resolvers: `dispatch_resolver.py` / `model_resolver.py` / `reasoning_resolver.py`. Full diagrams: **README В§Pipeline autГґnomo**.

### 3.3 Pipeline label state machine (the operational core)
```
рџ†• ~workflow:nova в†’ рџ”Ќ ~workflow:em_revisao в”Ђв”¬в”Ђ CLARO в†’ вњ… ~workflow:revisada
                                             в”‚            в”њв”Ђ intent в†’ рџ§© ~workflow:decomposta (architect opens N derived)
                                             в”‚            в””в”Ђ code   в†’ рџљЂ ~workflow:em_implementacao в†’ рџ“¬ ~workflow:em_pr
                                             в””в”Ђ VAGO в†’ рџЏ·пёЏ refinar + (рџ§  em_refinamento [intentв†’analyst]
                                                                      в”‚  or рџЏ›пёЏ em_arquitetura [feat/bug/refactorв†’architect/debugger])
                                                      в†• вЏёпёЏ ~workflow:aguardando_stakeholder (human removes to lift)
                                                      в†’ back to ~nova (up to 5 rounds, durable via ~refine:N в†’ в›” ~workflow:bloqueada)
```
- **Refinement gate** (persona by type: `intentв†’analyst`, `feature`/`refactor`в†’`architect`, `bugв†’debugger`); VAGO issues refined up to **5 rounds** (counter in `~refine:N`, survives restarts); high-impact gaps pause in `aguardando_stakeholder`. Passing intents are **decomposed** by `architect` (anti-flood: ONE derived issue + checklist). `max_parallel=2`, worker 2 replicas.
- **PR labels:** `~review:pendente в†’ em_andamento в†’ concluida`. Locks: `~batch:<sha8>` (**only when `shard_count>1`**), `~by:<id>` (owner). Markers: `~mention:processado` (idempotency), `~workflow:bloqueada` (hard block, **excludes from auto-resume** вЂ” human removes).
- **Mention/assignment routing** (`process_mentions`): issue+assignee/body в†’ inject `~workflow:nova`; **any** PR trigger в†’ unified `pr_unified` brief (worker reads real PR state, acts); issue+comment в†’ one-shot under `developer`. Anti-eco: comments by `gh_login` dropped. `CommentRef.is_pr_comment` routes PR comments.
- **Resume vs reaper (separate paths/counters):** *resume* continues paused work (`resume_max_attempts=10`); *reaper* re-claims **stale** claims (`em_implementacao`/`em_andamento` past ~45 min) via `~attempt:N` (`reaper_max_attempts=3`) вЂ” at attempt 3 **blocks** (`~workflow:bloqueada`) instead of releasing. Resume: fresh by default; only with `resume=True` **and** a `DispatchLedger` entry; over-budget `claude -p` session **promoted to fresh**. Recurring `WORKER_AUTH_EXPIRED` в†’ backoff `min(60В·2^n, 1800)s`/target after 3 fails.
- **One tick (`PipelineMonitor.tick`):** **reaper FIRST**, then **reconcile-before-dispatch** (completed fire-and-forget frees capacity same tick, #373) + **resume before new work**: reconcile в†’ classify в†’ review в†’ refine в†’ **resume** в†’ implement в†’ decompose в†’ pr_review в†’ pr_triage в†’ mentions. Schedule file (cron windows) wins if present, else every stage every tick.
- **Priority & triage:** `~prioridade:0..3` **reorder** candidate selection (lower first; unlabeled last; PRs inherit from the linked issue). **PR triage** labels `~review:pendente` only on branches this monitor owns (`auto/issue-*`, or `<prefix>/` when sharded), OR any PR if `enable_review_human_prs=True` (default **False**) вЂ” empty `head_ref` (cross-repo/fork) **silently skipped**.
- **Quality gate (NOT what you'd assume):** the **PR/review brief** (`_WORKER_PR_BRIEF`) runs the **full suite** `python3 -m pytest deile/tests/ -q` before approve/merge. The **implement briefs** (`implement`/`implement_resume`) run **only the impacted subset** `pytest <impacted> -p no:cov -q` and **forbid** the full suite вЂ” that's the reviewer's job (`_IMPACT_TEST_STRATEGY`, `briefs.py:59-67`). Full-suite has **no coverage gate** (no `--cov-fail-under` in `pytest.ini`). The merge gate is **prompt-enforced by the agent**, not deterministic CI. All briefs: `deile/orchestration/pipeline/briefs.py`.

**Pipeline package map** (`deile/orchestration/pipeline/`): `monitor.py` (tick + config), `stages.py` (per-stage + mention routing), `briefs.py` (**all** prompts), `labels.py` (state machine), `dispatch_ledger.py` (anti-double-dispatch), `{dispatch,model,reasoning}_resolver.py`, `follow_up_detector.py` + `post_merge_callback.py` (episodic memory per merged PR), `identity.py` (`MonitorIdentity` + sharding + `~by:<id>`), `claude_dispatcher.py`, `cost_estimator.py`, `reset.py`, `worktree_manager.py`.

> **Label edits use REST, NOT `gh issue/pr edit --add-label`** (that runs a GraphQL `login` query needing `read:org`, which the pipeline token lacks). Forge layer does REST in `forge/github_forge.py`. Manual:
> ```bash
> gh api -X POST   "repos/elimarcavalli/deile/issues/<N>/labels" -f 'labels[]=~workflow:revisada'
> gh api -X DELETE "repos/elimarcavalli/deile/issues/<N>/labels/%7Eworkflow%3Arevisada"   # ~=%7E :=%3A
> ```

## 4. Operational quick reference (local CLI)

| Task | Command |
|---|---|
| Run the agent | `python3 deile.py` |
| Run tests (summary only) | `python3 -m pytest deile/tests/ -q 2>&1 \| tail -5` |
| Single test | `python3 -m pytest deile/tests/path/to/test_x.py -v` |
| Coverage (opt-in вЂ” **not enforced**) | `python3 -m pytest deile/tests/ --cov=deile --cov-report=term` |
| Lint / imports / complexity | `ruff check deile/` В· `isort --check-only deile/` В· `radon cc deile/ -a` *(external tools вЂ” not pinned deps; install if missing)* |
| Multi-seed ordering check | `python3 -m pytest deile/tests/ -q --randomly-seed=<s>` for seeds `0,1,2,42,last` (`pytest-randomly`) |

**Testing gotchas:** `pytest.ini` uses `--strict-markers` (markers: `unit, integration, security, orchestration, bash, ui, slow, perf, e2e, e2e_{discord,telegram,whatsapp,meta}_live, manual, llm`) + `asyncio_mode=auto` (no `@pytest.mark.asyncio`). **No `--cov-fail-under`** вЂ” the 80% gate is not enforced. The **full suite carries pre-existing failures** that pass in isolation (ordering pollution) вЂ” diff against a baseline before declaring a regression. `deile/tests/` mixes pytest (`test_*.py`) with **standalone scripts** (`*_test.py`, `smoke_test_*.py`; run via `python deile/tests/all.py [--filter <sub>]`). Token-spending tests in `deile/tests/might/`. **Changed a signature?** `grep -rn` the whole test tree for every mock/`monkeypatch`/`patch`/fake and update each (stale double в†’ `TypeError` в†’ 500 в†’ misleading red), then run the **full** suite.

### 5.4 claude-worker вЂ” dispatching `claude -p` in-cluster
- **Setup:** `deploy.py k8s claude-login` captures host `~/.claude/` creds в†’ Secret `claude-credentials` в†’ initContainer copies to PVC `/home/claude/.claude/credentials.json` mode `0600` (refresh in-pod under flock). `--switch` = new account; `--no-interactive` = CI-safe.
- **Panel `[d]` в†’ `DispatchMatrixView`** вЂ” one row per stage, **6 editable columns**: Worker Г— Model Г— Timeout(s) Г— Retries Г— Cost-cap (USD/run) Г— Reasoning (Model anthropic-only when Worker=`claude-worker`). Hotkeys: `[enter]` picker, `[r]` reset, `[L]` switch login, `[I]` install, `[U]` uninstall, `[s]` scale, `[c]` cleanup (preview+confirm via `kubectl exec`), `[p]` `max_parallel` (`[a]`=`auto`), `[J]` JSONL retention.
- **Cost ledger (#445):** `claude -p` transcripts couple `--resume` continuity (bulky, ephemeral) with cost audit (tiny, permanent). `_do_cleanup` (startup hook + CronJob `claude-worker-cleanup` 03:00 UTC + `POST /v1/cleanup`) **harvests each orphan session's cost into a durable JSONL ledger BEFORE pruning** (`infra/k8s/jsonl_cost.py` = single pricing source; dedup by `session_id`); `session_tokens_audit.py` reads ledger + live JSONL, same pricing. **NEVER prune without harvesting** (incident: 337 transcripts deleted).

#### Instalar um CLI worker OAuth вЂ” `k8s cli-worker-login <kind>`
A maioria dos workers da frota autentica por chave de API (env) e sobe via `cli-worker-install <kind>`. Workers **oauth-capable** вЂ” aqueles cujo adapter declara um `OAuthSpec` (atributo `oauth` nГЈo-None), **independente do `auth_mode` default** вЂ” usam OAuth/device-auth em vez de chave, e sobem via `cli-worker-login` (o espelho genГ©rico do `claude-login`). Dois casos cabem aqui: adapters `auth_mode="oauth_file"` (claude-like) e adapters env-default mas oauth-capable como o **codex** (`auth_mode="env"` + `OAuthSpec`), cujo OAuth Г© **opt-in** (`DEILE_CODEX_AUTH=oauth`). Instalar o codex por este verb o coloca em **modo OAuth**: o manifest Г© renderizado com os blocos OAuth (PVC + initContainer + mount da `auth.json`) e o env `DEILE_CODEX_AUTH=oauth` Г© setado no Deployment. Workers SEM `OAuthSpec` sГЈo recusados (vГЈo para `cli-worker-install`).

```bash
python3 infra/k8s/deploy.py k8s cli-worker-login codex            # instala o codex-worker via OAuth (CLI no host)
python3 infra/k8s/deploy.py k8s cli-worker-login codex --switch   # forГ§a novo device-auth (trocar conta)
python3 infra/k8s/deploy.py k8s cli-worker-login codex --no-interactive  # CI: falha se a credencial nГЈo estiver no host
python3 infra/k8s/deploy.py k8s cli-worker-login codex --in-pod   # device-auth DENTRO do pod (operador SEM o CLI no host)
```

**Host-capture (sem `--in-pod`):** detecta a credencial OAuth no host no path do `OAuthSpec` (`~/.codex/auth.json`, ou `$CODEX_HOME/auth.json`); **se ausente E interativo, roda o `login_cmd` do adapter (`codex login --device-auth`) no host** вЂ” o operador completa o device-auth no browser; captura a credencial, cria o Secret, gera+aplica o manifest (modo OAuth), seta `DEILE_<KIND>_AUTH=oauth` e escala a 1.

**`--in-pod` (operador sem o CLI no host):** aplica Secret placeholder + manifest OAuth para o pod subir; roda o `login_cmd` **dentro do pod** via `kubectl exec -it` вЂ” o pod imprime a URL + cГіdigo de device-auth, o operador autoriza no browser; lГЄ a `auth.json` gerada no `CODEX_HOME` do pod, atualiza o Secret de credencial com o valor real e faz rollout restart (o initContainer copia a credencial real ao PVC mode `0600`).

Ambos os modos: idempotentes; **nunca logam o conteГєdo da credencial** (sГі o comprimento). **Quando usar OAuth vs env:** OAuth para consumir a assinatura ChatGPT do codex (o `gpt-5-codex` via Responses API retorna 401 com project API key вЂ” exige `codex login`); env (`OPENAI_API_KEY` via `cli-worker-install`) para automaГ§ГЈo robusta sem expiraГ§ГЈo. MГіdulos: `infra/k8s/_cli_worker_login.py` (host-capture) + `_k8s_in_pod_cli_worker_login` em `deploy.py` (in-pod); render OAuth em `_cli_worker_gen.render_manifests(..., oauth_mode=True)`.

### 5.5 deile-monitor вЂ” the 24/7 cluster supervisor (6th pod)
Two-phase deterministic tick (PR #504): **Phase A** = mechanical sweep, **zero LLM** вЂ” 8 "vigias" (OAuth health, error pods, orphan issues, `auto/*` PR attempt N/3, awaiting-stakeholder, failed Jobs, pipeline-pod health, follow-ups); **Phase B** invokes the `monitor` persona **only** when Phase-A candidates survive (via state file). Steady state = **no tokens** (was ~3.5M/tick as a tool-loop вЂ” #504 fix). Tick 30 min; RBAC `deile-monitor-sa` (read pods/jobs + `pods/exec` for OAuth renew). Persona-driven (`personas/instructions/monitor.md`, hot-reload). Main process `monitor_command_server.py` (`args:`, tini-wrapped) runs the tick as a subprocess AND serves a Bearer control plane on `:8769` (status, `pause`/`resume`/`ack`/`force-tick`, read-only Q&A via `monitor_qa` in-pod вЂ” the only pod seeing kubectl + forge + `/state`). `deilebot` `/monitor` cog + cluster-question auto-route consume it; `/v1/health` 503 on stale tick в†’ restart.

### 5.6 Observability gotchas вЂ” what you see is NOT always reality
1. **`ps`/`pgrep` work** (`procps` in the image). Old image: `cat /proc/*/cmdline | tr '\0' ' '` (`/proc` shows the host PIDNS).
2. **Pipeline logs look empty when idle** вЂ” the monitor emits per-tick; short tails get evicted by health probes. Use `--tail=500` / `--since=10m`. Unbuffered (`PYTHONUNBUFFERED=1` + tini), so true silence-with-RUNNING = deadlock (rare) or too-aggressive a tail.
3. **`.lease.json` mtime is NOT liveness for `claude -p`** вЂ” the wrapper rewrites `heartbeat_at` every 5 s regardless, lease `pid` = **wrapper** PID. Each dispatch writes `claude_pid`; `_find_active_lease` reports `claude_running` (True iff that PID alive). **Trust `claude_running` (or `pgrep -a claude`), never lease mtime.**
4. **OAuth renew is an operational trap.** The claude-worker access token expires (~hours); the in-pod "refresh" does **not** rotate the refresh token в†’ the whole fleet falls into `401 / WORKER_AUTH_EXPIRED`, looking like a "stuck pipeline" with no code cause. Fix from host: **`python3 infra/k8s/deploy.py k8s claude-renew`** (reads the macOS keychain). Check this FIRST when dispatches mysteriously stall.

## 8. Task playbooks (recipes for working autonomously)

- **Ship a code change to the cluster** в†’ edit в†’ `deploy.py k8s build --restart --yes` (image is baked; **no rebuild = no effect**) в†’ confirm with `k8s status` + `k8s logs`. Config-only change to the pipeline pod в†’ `kubectl set env` + `k8s restart` avoids a rebuild.
- **"Pipeline looks stuck"** в†’ (1) **OAuth first** вЂ” `deploy.py k8s claude-renew` from the host (expired token masquerades as stuck, В§5.6); (2) liveness via `claude_running` / `pgrep -a claude`, never lease mtime; (3) `k8s logs deile-pipeline --tail=500`; (4) panel `[t]` tokens or status server `/v1/pipeline-status`. `/babysit-deile` arms watchers for exactly this.
- **Work on a brief / the quality gate** в†’ briefs in `briefs.py`; remember implement=subset, PR=full suite, no cov gate; after editing run `pytest deile/tests/orchestration/pipeline/test_briefs* -v` (invariants test-guarded).
- **Tune per-stage routing/model/reasoning** в†’ panel `[d]` matrix, or env `DEILE_PIPELINE_{DISPATCH,MODEL,REASONING}_<STAGE>` (rollout `deile-pipeline`, no rebuild).

## 9. Hard-won operational lessons (don't re-pay the cost)

- **claude-worker resume context measurement bug (fixed `065e3ea`):** the gate summed `cache_read` across all rounds (11.5M in a ~70K session) vs a fixed 100K ceiling в†’ every long review restarted. Fix = peak-of-1-round + 80% of the real window. `claude -p` has **no tunable auto-compact** (only via SDK). Long sessions restarting в†’ suspect context-measurement, not the model.
- **`ultracode` on claude-worker** needs `claude в‰Ґ 2.1.158` (pinned in the Dockerfile) and injects the keyword "workflow" + `xhigh`. Bumping the pin can hit a **Docker layer-cache** trap вЂ” force a clean build layer.
- **`infra/k8s/` packaging + destructive-code fail-safe:** the module ships in the image via explicit `COPY` + a `.dockerignore` exception. **Never prune without harvesting first**, and always test the `None` path вЂ” the cost-ledger incident (337 transcripts deleted) came from pruning before harvesting.
- **Verify before asserting:** never claim "works / it's live" without empirical proof; say what you couldn't test. (`feedback_honesty_verify`.)
- **`/proc`-first skepticism** for claude-worker liveness; run a skeptical delta-loop, not a hopeful one (`/babysit-deile` encodes this).
- More incident detail in persistent memory (`MEMORY.md`) вЂ” cost accounting, loop-guards history, refinement-gate/state-machine deep notes.

## 10. Gotchas (not in the system design)

- **At least one provider key is required at startup** (`ANTHROPIC_/OPENAI_/DEEPSEEK_/GOOGLE_API_KEY`); `bootstrap_providers()` (`core/models/bootstrap.py`) does conditional registration.
- **`deilebot` is a separate repo** (`elimarcavalli/deilebot`). `pip install -e ".[bot]"` (git-URL extra) for cloud/CI; clone it + `pip install -e ./deilebot` to hack on the bot. `messaging.discord_*` tools auto-register **only** when `import deilebot` succeeds **and** `DEILE_BOT_ENDPOINT` + `DEILE_BOT_AUTH_TOKEN` are set. Run the bot from the `deile/` repo root.
- **Touching messaging tools?** Open `08-SEGURANCA.md` first вЂ” DM and role-mention tools are `SecurityLevel.DANGEROUS`, gated by `ApprovalSystem` (`orchestration/approval_system.py`); that gate is non-trivial.
- **`Settings` is a singleton** вЂ” `from deile.config.settings import get_settings`, never `Settings()`.
- **Personas are MD-driven** вЂ” edit `personas/instructions/*.md`; no Python change.
- **`CLAUDE.md` is versioned** вЂ” nothing in `.gitignore` ignores it (lists `.claude/`, `.openclaude/*`, `.claude-worktrees/` вЂ” **no** `*claude*` glob nor `!CLAUDE.md` negation; don't go hunting for one).
- **Discord interactions** в†’ use native modals/buttons/selects; check the docs before implementing.
- **The tool registry does NOT enforce permissions/audit.** "Permission-before-action + typed audit on every tool" is **not** in the central exec path (`tool_loop_executor`) вЂ” only `bash_tool` checks `PermissionManager` (and allows-all when no manager injected); other tools run ungated (DecisГЈo #29 deferred). The reliably-firing audit event is the **loop-guard abort** (`SUSPICIOUS_ACTIVITY`). `secrets_scanner` redaction IS wired in the session store (`core/session_store.py`), covering GH/GL/AWS/bearer/private-key patterns. Rely on loop guard + NetworkPolicy + tool whitelist for containment, not a per-tool gate that mostly isn't there.
