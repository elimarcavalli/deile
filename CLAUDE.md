# DEILE — Claude Code Context

> **What DEILE is, in one breath:** a multi-provider **agent harness** (turns an LLM API into a real tool-using agent) wrapped by an **autonomous orchestration platform**. Two faces, one Python core: (1) interactive **CLI** agent; (2) autonomous **Kubernetes pipeline** monitoring a GitHub **or** GitLab repo — refines issues, implements on `auto/issue-N` branches, opens PRs/MRs, reviews them, no human in the loop. Drives **its own engine** (`deile-worker`) **or the Claude Code CLI** (`claude-worker`, `claude -p`) per stage.

**Loaded into every session** — the architecture map + how-to-work guide; **only verified truth**. Product reference = `README.md` (code-validated, on `main`) — read it **semantically on demand** (§1), never whole.

### Operating rules
1. **The code is the source of truth.** `Comentário não é prova, o código é.` State how something works → cite `arquivo:linha`; no source → mark it a supposition. Never guess.
2. **Any doubt → `README.md` section and/or pillar doc, then the code.** Don't fill gaps with assumptions; declare confidence, separate verified from supposed.
3. **Use skills/slash commands proactively** — don't reinvent what a command does. High-value: `/REVIEW-PR` · `/REVIEW-CETICA` (skeptical) · `/ISSUE-REFINE` · `/EVOLVE` (doc-vs-code gap) · `/SCHEDULE-PIPELINE` · `/babysit-deile` (claude-worker watcher) · `/DAILY-STANDUP` · `/REPO-STORY` · `/PORT-SCAN` · `/COMMIT` · `/PRE-COMPACT`; `superpowers:*` (brainstorming, test-driven-development, systematic-debugging, verification-before-completion); `code-review` / `simplify`. List via the Skill tool; if one fits even at low probability, invoke it.
4. **Dispatching subagents → ALWAYS tell each to read `CLAUDE.md` first** (+ the specific `README.md` section + pillar doc for its slice). Never spawn a context-blind agent here. Parallelize 2+ independent, no-shared-state tasks.
5. **Alignment before autonomous go** on non-trivial tasks: sketch plan + risks, get the nod (a ready, well-specified plan can proceed directly). Quality > workaround — understand impacts, reuse the architecture, refactor when needed (with prior alignment), never duplicate logic. Self-critique before closing ("where does this break?").
6. **Conventions** (full rules in global `~/.claude/CLAUDE.md`): commit titles in English `type(scope): description`, body in PT-BR; **never** add `Co-Authored-By: Claude` or "Generated by Claude Code"; all user-visible docs/comments in correct **PT-BR**; prefer "Humano". `elimarcavalli` on GitHub = the Human; `deile-one` = the bot — don't assume authorship. SQL scripts are the Human's to run.

## 2. Mandatory protocol (run before every non-trivial turn)

Before the first `Write`, `Edit`, or mutating `Bash` of a turn:
1. Classify the **action** → trigger index in `03-PRINCIPIOS-ARQUITETURAIS.md`.
2. Classify the **target path(s)** → subpackage map in `02-ARQUITETURA.md` / §3.1 below.
3. Match the **user's keywords** → architecture/scope/capability terms point to the relevant pillar (`00-VISAO-GERAL.md`).
4. **Union** the pillars implied; `Read` every unread one **before** mutating.
5. If scope grows mid-task, **stop and re-run** the protocol.

**Exempt:** typos, whitespace, single-line cosmetic edits, renaming a strictly-local var, read-only questions, running tests/lint/formatters, editing `.env`/lockfiles/`CLAUDE.md`/`docs/system_design/`. When uncertain → **not** exempt.

## 3. Architecture map — the real system

### 3.1 The Python package (`deile/`)
Hexagonal, registry-driven, **async-first** (no sync I/O inside `async def`). Adding a tool/command/parser/persona/skill must **not** touch the core (Open/Closed). The core never imports external SDKs — adapters live in `infrastructure/` or concrete providers.

| Subpackage | Responsibility | | Subpackage | Responsibility |
|---|---|---|---|---|
| `core/` | Agent loop, `ToolLoopExecutor` | | `orchestration/` | Plans, workflows, **pipeline/**, **forge/**, approval |
| `core/models/` | Providers, `ModelRouter`+`TierRouter`, streaming, `reasoning.py` | | `runtime/` | Per-process state file, Unix-socket status server, fleet registry (#303) |
| `events/` | Async `EventBus` + handlers | | `observability/` | OTLP tracer/metrics, dispatch-span adapter (#303/#48) |
| `tools/` | Tool registry + impls (+ `messaging/`) | | `cron/` | `CronStore` (SQLite) + `CronRunner` |
| `commands/` | Slash commands (`builtin/`) + registry | | `memory/` | 4 layers (working/episodic/semantic/procedural) |
| `parsers/` | Input parsing (files, diffs, refs, commands) | | `security/` | `PermissionManager`, `AuditLogger`, `SecretsScanner` |
| `personas/` | MD instructions + YAML library | | `storage/` | logger, `UsageRepository`+`BudgetGuard`, embeddings |
| `skills/` | Discovery, registry, hot-reload | | `ui/` | Rich streaming, sub-agent panel, `ui/panel/` |
| `integrations/` | HTTP client for the reverse `agent → bot` arrow | | `infrastructure/` | external adapters (SDKs) |
| `evolution/` | auto-learning (**experimental**) | | `plugins/` | plugin manager, hot-reload (**no real sandbox**) |
| `preferences/` | backing store for the `*_preference` tools | | `config/` | `Settings` singleton, YAML, profiles |
| `log_mgmt/` | log analysis/rotation/dispatch | | | |

> **Entry point:** `python3 deile.py` is a thin launcher (prepares venv) → `deile.cli.main`; the interactive shell class is **`_DeileCLI`** in `deile/cli.py` (NOT `DeileAgentCLI`). All logic lives under `deile/`.

**Inventories — verify in code, never hard-code:** tools = `Tool` subclass `name` in `deile/tools/**` + `DEFAULT_TOOL_PACKAGES` (`discovery.py`); commands = `deile/commands/builtin/*_command.py`; models/tiers = `model_providers.yaml`; personas = `deile/personas/{library/*.yaml,instructions/*.md}` (now: `analyst`, `architect`, `debugger`, `developer`, `reviewer` + instr. `discord_developer`, `monitor`, `monitor_qa`, `fallback`); skills = 6 scan dirs. Real set in §3.6–3.8.

**A CLI turn (core loop):** `_DeileCLI` → `DeileAgent`; parsers extract file/command refs; slash commands dispatch via `CommandRegistry`, else `ModelRouter`/`TierRouter` picks provider/model; the provider streams `UnifiedStreamEvent`s (`TEXT_DELTA`, `TOOL_USE_START`/`TOOL_USE_END`, …); on a tool-use `ToolLoopExecutor` runs it and feeds the `ToolResult` back (cap `max_tool_iterations`=100, `DEILE_MAX_TOOL_ITERATIONS`); `StreamingRenderer` paints via `rich.live.Live`; `EventBus` publishes telemetry/persona/tool events. Streaming-first (`process_input_stream`) is default.

### 3.2 The worker fleet (per-stage selectable execution engines)
- **`deile-worker`** — **DEILE Python in-process** (`wrapper.py worker` → `infra/k8s/worker_server.py`); DEILE's own provider routing.
- **`claude-worker`** — **`claude -p`** (Claude Code CLI) in an isolated **workdir per task_id** (`gh repo clone` → `/home/claude/work/<task_id>/repo`); OAuth creds on a PVC.
- **CLI worker fleet (Decisão #51)** — 5 pluggable coding CLIs, each a pod with its own Service: **`opencode-worker` :8771 · `codex-worker` :8772 · `qwen-worker` :8773 · `aider-worker` :8774 · `goose-worker` :8775**. One generic server (`infra/k8s/cli_worker_server.py` over `_worker_core.py`); each CLI is plugged by **one adapter** `infra/k8s/cli_adapters/<kind>.py` satisfying the `CliAdapter` Protocol — auto-discovery builds `ADAPTERS` (single source of truth driving the resolver, panel, manifest/NetworkPolicy gen). Born `replicas:0` (scale-to-zero); scaled 0→1 on demand by `cli_worker_scaler.py`. Adding a worker = **one adapter, no consumer edited** (§5.8, §3.5). (Antigravity exists only as a documented GATE — not active.)

Engine picked **per stage** via `dispatch_resolver.py` (`VALID_DISPATCHERS` derived from `ADAPTERS` + the two built-ins). **5 stages × 3 axes**, each with a global fallback + env/settings persistence:

| Stage | Worker (`dispatch`) | Model | Reasoning |
|---|---|---|---|
| `classify` `refine` `implement` `pr_review` `follow_ups` | `DEILE_PIPELINE_DISPATCH_<STAGE>` → `DEILE_PIPELINE_DISPATCH_MODE` (default `deile-worker`) | `DEILE_PIPELINE_MODEL_<STAGE>` → `DEILE_PREFERRED_MODEL` | `DEILE_PIPELINE_REASONING_<STAGE>` → `DEILE_REASONING_EFFORT` |

Resolvers: `dispatch_resolver.py` / `model_resolver.py` / `reasoning_resolver.py`. Full diagrams: **README §Pipeline autônomo**.

### 3.3 Pipeline label state machine (the operational core)
```
🆕 ~workflow:nova → 🔍 ~workflow:em_revisao ─┬─ CLARO → ✅ ~workflow:revisada
                                             │            ├─ intent → 🧩 ~workflow:decomposta (architect opens N derived)
                                             │            └─ code   → 🚀 ~workflow:em_implementacao → 📬 ~workflow:em_pr
                                             └─ VAGO → 🏷️ refinar + (🧠 em_refinamento [intent→analyst]
                                                                      │  or 🏛️ em_arquitetura [feat/bug/refactor→architect/debugger])
                                                      ↕ ⏸️ ~workflow:aguardando_stakeholder (human removes to lift)
                                                      → back to ~nova (up to 5 rounds, durable via ~refine:N → ⛔ ~workflow:bloqueada)
```
- **Refinement gate** (persona by type: `intent→analyst`, `feature`/`refactor`→`architect`, `bug→debugger`); VAGO issues refined up to **5 rounds** (counter in `~refine:N`, survives restarts); high-impact gaps pause in `aguardando_stakeholder`. Passing intents are **decomposed** by `architect` (anti-flood: ONE derived issue + checklist). `max_parallel=2`, worker 2 replicas.
- **PR labels:** `~review:pendente → em_andamento → concluida`. Locks: `~batch:<sha8>` (**only when `shard_count>1`**), `~by:<id>` (owner). Markers: `~mention:processado` (idempotency), `~workflow:bloqueada` (hard block, **excludes from auto-resume** — human removes).
- **Mention/assignment routing** (`process_mentions`): issue+assignee/body → inject `~workflow:nova`; **any** PR trigger → unified `pr_unified` brief (worker reads real PR state, acts); issue+comment → one-shot under `developer`. Anti-eco: comments by `gh_login` dropped. `CommentRef.is_pr_comment` routes PR comments.
- **Resume vs reaper (separate paths/counters):** *resume* continues paused work (`resume_max_attempts=10`); *reaper* re-claims **stale** claims (`em_implementacao`/`em_andamento` past ~45 min) via `~attempt:N` (`reaper_max_attempts=3`) — at attempt 3 **blocks** (`~workflow:bloqueada`) instead of releasing. Resume: fresh by default; only with `resume=True` **and** a `DispatchLedger` entry; over-budget `claude -p` session **promoted to fresh**. Recurring `WORKER_AUTH_EXPIRED` → backoff `min(60·2^n, 1800)s`/target after 3 fails.
- **One tick (`PipelineMonitor.tick`):** **reaper FIRST**, then **reconcile-before-dispatch** (completed fire-and-forget frees capacity same tick, #373) + **resume before new work**: reconcile → classify → review → refine → **resume** → implement → decompose → pr_review → pr_triage → mentions. Schedule file (cron windows) wins if present, else every stage every tick.
- **Priority & triage:** `~prioridade:0..3` **reorder** candidate selection (lower first; unlabeled last; PRs inherit from the linked issue). **PR triage** labels `~review:pendente` only on branches this monitor owns (`auto/issue-*`, or `<prefix>/` when sharded), OR any PR if `enable_review_human_prs=True` (default **False**) — empty `head_ref` (cross-repo/fork) **silently skipped**.
- **Quality gate (NOT what you'd assume):** the **PR/review brief** (`_WORKER_PR_BRIEF`) runs the **full suite** `python3 -m pytest deile/tests/ -q` before approve/merge. The **implement briefs** (`implement`/`implement_resume`) run **only the impacted subset** `pytest <impacted> -p no:cov -q` and **forbid** the full suite — that's the reviewer's job (`_IMPACT_TEST_STRATEGY`, `briefs.py:59-67`). Full-suite has **no coverage gate** (no `--cov-fail-under` in `pytest.ini`). The merge gate is **prompt-enforced by the agent**, not deterministic CI. All briefs: `deile/orchestration/pipeline/briefs.py`.

**Pipeline package map** (`deile/orchestration/pipeline/`): `monitor.py` (tick + config), `stages.py` (per-stage + mention routing), `briefs.py` (**all** prompts), `labels.py` (state machine), `dispatch_ledger.py` (anti-double-dispatch), `{dispatch,model,reasoning}_resolver.py`, `follow_up_detector.py` + `post_merge_callback.py` (episodic memory per merged PR), `identity.py` (`MonitorIdentity` + sharding + `~by:<id>`), `claude_dispatcher.py`, `cost_estimator.py`, `reset.py`, `worktree_manager.py`.

> **Label edits use REST, NOT `gh issue/pr edit --add-label`** (that runs a GraphQL `login` query needing `read:org`, which the pipeline token lacks). Forge layer does REST in `forge/github_forge.py`. Manual:
> ```bash
> gh api -X POST   "repos/elimarcavalli/deile/issues/<N>/labels" -f 'labels[]=~workflow:revisada'
> gh api -X DELETE "repos/elimarcavalli/deile/issues/<N>/labels/%7Eworkflow%3Arevisada"   # ~=%7E :=%3A
> ```

### 3.4 Other subsystems — working notes
- **Reasoning effort** (`core/models/reasoning.py`, single source, fail-open): vocab `low/medium/high/xhigh/max/ultracode/auto` (Claude-Code style) → each provider's native param. Global `DEILE_REASONING_EFFORT` / `/reasoning`; per-stage above. claude-worker: `ultracode` → `xhigh` + prompt preamble (needs `claude ≥ 2.1.158`).
- **Skills** — pure-Markdown, no Python. 6 scan dirs (low→high: bundled `deile/skills/library/**` < `~/.deile/skills` < `~/.claude/commands` < `<cwd>/.deile/skills` < `<cwd>/.claude/commands` < configured). 4 auto-inject triggers (`file_globs`/`code_block_langs`/`keywords`(escaped)/`file_content_patterns`(4 KiB, root-contained)). Hot-reload ~0.5 s. Bundled: `python`, `typescript`, `tdd`. Also `invoke_skill`/`list_skills`/`/<name>`.
- **Sub-DEILEs** — `dispatch_parallel_subagents` splits a request into 2–5 independent sub-tasks (clean sessions, live panel). `LocalSubAgentRunner` (default, in-process) or `WorkerSubAgentRunner` (`DEILE_SUBAGENT_RUNNER=worker` → `deile-worker`). Tunables: `DEILE_SUBAGENT_MAX_PARALLEL` (3), `DEILE_SUBAGENT_BUDGET_S` (600).
- **Loop guard** (`core/loop_guard.py`, wired into the tool-loop **and** all 3 non-streaming providers) aborts a runaway turn via 4 rules: hard ceiling (`loop_guard_max_calls=50`), ≥3 identical consecutive `(name,args)`, ≥3-of-5 sliding window, ≥6 no-progress (empty/error) — emits `AuditEventType.SUSPICIOUS_ACTIVITY`. Tunables: `DEILE_LOOP_GUARD_*` / `DEILE_MAX_TOOL_ITERATIONS` (100, distinct from the 50 ceiling).
- **OTLP names** (debug ref) — spans `deile.turn`, `deile.tool.<name>`, `deile.llm.call`, root `deile.dispatch` (+ events `dispatch.received/model_resolved/progress/tool_burst/completed/failed`, child `git.commit`/`git.push`/`forge.pr_open`/`forge.pr_review`); metrics `deile.tokens.total`, `deile.cost.usd.total`, `deile.tool.duration_ms`, `deile.turn.duration_ms`, `deile.errors.total`. No secrets in spans; `session_id` not a label.
- **Memory** — 4 layers via `MemoryManager`; episodic = SQLite (`aiosqlite`, `episodes.db`). Never store secrets/PII; always `await`; **open the layer module before calling — signatures evolve**, don't invent method names.
- **Runtime + observability** — each process publishes `~/.deile/run/<id>.json` (atomic + heartbeat) + a Unix-socket status server; `registry.json` (flock, GC dead PIDs) = fleet view. OTLP (`DEILE_OTLP_*`, no-op when endpoint empty or `DEILE_OBSERVABILITY_DISABLED=true`).

## 4. Operational quick reference (local CLI)

| Task | Command |
|---|---|
| Run the agent | `python3 deile.py` |
| Run tests (summary only) | `python3 -m pytest deile/tests/ -q 2>&1 \| tail -5` |
| Single test | `python3 -m pytest deile/tests/path/to/test_x.py -v` |
| Coverage (opt-in — **not enforced**) | `python3 -m pytest deile/tests/ --cov=deile --cov-report=term` |

**Testing gotchas:** `pytest.ini` uses `--strict-markers` (markers: `unit, integration, security, orchestration, bash, ui, slow, perf, e2e, e2e_{discord,telegram,whatsapp,meta}_live, manual, llm`) + `asyncio_mode=auto` (no `@pytest.mark.asyncio`). **No `--cov-fail-under`** — the 80% gate is not enforced. The **full suite carries pre-existing failures** that pass in isolation (ordering pollution) — diff against a baseline before declaring a regression. `deile/tests/` mixes pytest (`test_*.py`) with **standalone scripts** (`*_test.py`, `smoke_test_*.py`; run via `python deile/tests/all.py [--filter <sub>]`). Token-spending tests in `deile/tests/might/`. **Changed a signature?** `grep -rn` the whole test tree for every mock/`monkeypatch`/`patch`/fake and update each (stale double → `TypeError` → 500 → misleading red), then run the **full** suite.

## 5. Kubernetes / cluster operations

Cluster = **Rancher Desktop (k3s/containerd)**, single image **`deile-stack:local`** (`imagePullPolicy: Never`). `/app` is **baked at build time** (`COPY`, not mounted) → **code changes live only after rebuild + restart**. `kubectl` at `~/.rd/bin/kubectl` (may not be on `PATH`). ### 5.3 Multi-namespace — DEILE runs many concurrent stacks
One stack per namespace (default `deile`; `deile-gl` = GitLab pilot, usually scaled to 0). **Always survey first** + **always pass the namespace flag** — a bare `kubectl apply -f` lands in `default` and pollutes it.

The 6 core deployments are namespace-agnostic; **caveat:** some auxiliary manifests (NetworkPolicy, claude-worker, certain PVCs/CronJobs) hardcode `namespace: deile` — full for core, partial for auxiliaries. Never touch `default`, `kube-system`, `kube-public`, `kube-node-lease`.

### 5.4 claude-worker — dispatching `claude -p` in-cluster
- **Setup:** `deploy.py k8s claude-login` captures host `~/.claude/` creds → Secret `claude-credentials` → initContainer copies to PVC `/home/claude/.claude/credentials.json` mode `0600` (refresh in-pod under flock). `--switch` = new account; `--no-interactive` = CI-safe.
- **Panel `[d]` → `DispatchMatrixView`** — one row per stage, **6 editable columns**: Worker × Model × Timeout(s) × Retries × Cost-cap (USD/run) × Reasoning (Model anthropic-only when Worker=`claude-worker`). Hotkeys: `[enter]` picker, `[r]` reset, `[L]` switch login, `[I]` install, `[U]` uninstall, `[s]` scale, `[c]` cleanup (preview+confirm via `kubectl exec`), `[p]` `max_parallel` (`[a]`=`auto`), `[J]` JSONL retention.
- **Cost ledger (#445):** `claude -p` transcripts couple `--resume` continuity (bulky, ephemeral) with cost audit (tiny, permanent). `_do_cleanup` (startup hook + CronJob `claude-worker-cleanup` 03:00 UTC + `POST /v1/cleanup`) **harvests each orphan session's cost into a durable JSONL ledger BEFORE pruning** (`infra/k8s/jsonl_cost.py` = single pricing source; dedup by `session_id`); `session_tokens_audit.py` reads ledger + live JSONL, same pricing. **NEVER prune without harvesting** (incident: 337 transcripts deleted).

| claude-worker env var | Default | Use |
|---|---|---|
| `DEILE_CLAUDE_MAX_BUDGET_USD` | `8` | hard cost cap → `claude --max-budget-usd` per dispatch |
| `DEILE_CLAUDE_RESUME_TOKEN_BUDGET` | (unset) | absolute token override for resume→fresh; **when unset**, threshold = `DEILE_CLAUDE_RESUME_CONTEXT_FRACTION` (`0.80`) × model context window (over-budget `claude -p` → promoted to fresh) |
| `DEILE_CLAUDE_CLEANUP_RETENTION_DAYS` | `7` | workdir retention (startup + CronJob) |
| `DEILE_CLAUDE_JSONL_RETENTION_DAYS` | `30` | transcript retention (separate knob; panel `[J]`) |
| `DEILE_CLAUDE_JSONL_ORPHAN_GRACE_S` | `3600` | grace before pruning an orphan transcript (TOCTOU guard) |
| `DEILE_PIPELINE_MAX_PARALLEL` | `2` | max simultaneous dispatches (read by the **pipeline monitor**, not the worker) |

**Threat model:** ingress to `:8767` restricted to `deile-pipeline` only. **Egress is NOT host-whitelisted at L3/L4** — NetworkPolicy allows generic TCP 443 anywhere (`to: []`) + DNS (k3s has no FQDN-aware CNI). The host/repo allowlist (`api.anthropic.com`, `github.com`, `gitlab.com`, per-repo) is enforced **only in `wrapper.py`** via ConfigMap `claude-worker-allowed-repos` (a `git` guard). Known gap (spec §7): prompt injection could exfiltrate creds via legitimate channels (`docs/superpowers/specs/2026-05-26-claude-worker-design.md`). *(Decisão #43 mislabels this an L3/L4 whitelist — the real control is application-level.)*

#### Configurar provider de workers OpenAI-compatible (qwen etc.)
Workers OpenAI-compatible (qwen e afins) escolhem o **provider** pela tríade `OPENAI_BASE_URL` + `OPENAI_API_KEY` + `OPENAI_MODEL`. Em vez de `kubectl set env` manual (não reproduzível), declare-a no **`.env`** com a convenção `DEILE_CLI_<KIND>_ENV_<VARNAME>=<valor>` — `_cli_worker_gen.py` injeta cada uma como env var `<VARNAME>` no Deployment do worker `<kind>` ao renderizar o manifest, e `_cli_worker_install.py` propaga as sensíveis ao Secret. Sensível (casa `auth_env_keys` **ou** termina em `_API_KEY`/`_TOKEN`) → `secretKeyRef` no Secret `cli-worker-keys`, **nunca** literal no manifest; não-sensível (ex.: `OPENAI_BASE_URL`, `OPENAI_MODEL`) → valor literal. Ausência total das vars → comportamento atual inalterado (sem override). Aplica-se via `deploy.py k8s cli-worker-install <kind>` (ou re-render do manifest).

Três rotas (`<KIND>` = `QWEN`):
```dotenv
# Dashscope nativo
DEILE_CLI_QWEN_ENV_OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DEILE_CLI_QWEN_ENV_OPENAI_API_KEY=<key Dashscope>     # → Secret cli-worker-keys
DEILE_CLI_QWEN_ENV_OPENAI_MODEL=qwen3-coder-plus

# OpenRouter (barato)
DEILE_CLI_QWEN_ENV_OPENAI_BASE_URL=https://openrouter.ai/api/v1
DEILE_CLI_QWEN_ENV_OPENAI_API_KEY=<key OpenRouter>    # → Secret cli-worker-keys
DEILE_CLI_QWEN_ENV_OPENAI_MODEL=qwen/qwen3-coder

# OpenAI direto (sem base_url → default do CLI)
DEILE_CLI_QWEN_ENV_OPENAI_API_KEY=<key OpenAI>        # → Secret cli-worker-keys
DEILE_CLI_QWEN_ENV_OPENAI_MODEL=<modelo OpenAI>
```

#### Instalar um CLI worker OAuth — `k8s cli-worker-login <kind>`
A maioria dos workers da frota autentica por chave de API (env) e sobe via `cli-worker-install <kind>`. Workers **oauth-capable** — aqueles cujo adapter declara um `OAuthSpec` (atributo `oauth` não-None), **independente do `auth_mode` default** — usam OAuth/device-auth em vez de chave, e sobem via `cli-worker-login` (o espelho genérico do `claude-login`). Dois casos cabem aqui: adapters `auth_mode="oauth_file"` (claude-like) e adapters env-default mas oauth-capable como o **codex** (`auth_mode="env"` + `OAuthSpec`), cujo OAuth é **opt-in** (`DEILE_CODEX_AUTH=oauth`). Instalar o codex por este verb o coloca em **modo OAuth**: o manifest é renderizado com os blocos OAuth (PVC + initContainer + mount da `auth.json`) e o env `DEILE_CODEX_AUTH=oauth` é setado no Deployment. Workers SEM `OAuthSpec` são recusados (vão para `cli-worker-install`).

```bash
python3 infra/k8s/deploy.py k8s cli-worker-login codex            # instala o codex-worker via OAuth (CLI no host)
python3 infra/k8s/deploy.py k8s cli-worker-login codex --switch   # força novo device-auth (trocar conta)
python3 infra/k8s/deploy.py k8s cli-worker-login codex --no-interactive  # CI: falha se a credencial não estiver no host
python3 infra/k8s/deploy.py k8s cli-worker-login codex --in-pod   # device-auth DENTRO do pod (operador SEM o CLI no host)
```

**Host-capture (sem `--in-pod`):** detecta a credencial OAuth no host no path do `OAuthSpec` (`~/.codex/auth.json`, ou `$CODEX_HOME/auth.json`); **se ausente E interativo, roda o `login_cmd` do adapter (`codex login --device-auth`) no host** — o operador completa o device-auth no browser; captura a credencial, cria o Secret, gera+aplica o manifest (modo OAuth), seta `DEILE_<KIND>_AUTH=oauth` e escala a 1.

**`--in-pod` (operador sem o CLI no host):** aplica Secret placeholder + manifest OAuth para o pod subir; roda o `login_cmd` **dentro do pod** via `kubectl exec -it` — o pod imprime a URL + código de device-auth, o operador autoriza no browser; lê a `auth.json` gerada no `CODEX_HOME` do pod, atualiza o Secret de credencial com o valor real e faz rollout restart (o initContainer copia a credencial real ao PVC mode `0600`).

Ambos os modos: idempotentes; **nunca logam o conteúdo da credencial** (só o comprimento). **Quando usar OAuth vs env:** OAuth para consumir a assinatura ChatGPT do codex (o `gpt-5-codex` via Responses API retorna 401 com project API key — exige `codex login`); env (`OPENAI_API_KEY` via `cli-worker-install`) para automação robusta sem expiração. Módulos: `infra/k8s/_cli_worker_login.py` (host-capture) + `_k8s_in_pod_cli_worker_login` em `deploy.py` (in-pod); render OAuth em `_cli_worker_gen.render_manifests(..., oauth_mode=True)`.

### 5.5 deile-monitor — the 24/7 cluster supervisor (6th pod)
Two-phase deterministic tick (PR #504): **Phase A** = mechanical sweep, **zero LLM** — 8 "vigias" (OAuth health, error pods, orphan issues, `auto/*` PR attempt N/3, awaiting-stakeholder, failed Jobs, pipeline-pod health, follow-ups); **Phase B** invokes the `monitor` persona **only** when Phase-A candidates survive (via state file). Steady state = **no tokens** (was ~3.5M/tick as a tool-loop — #504 fix). Tick 30 min; RBAC `deile-monitor-sa` (read pods/jobs + `pods/exec` for OAuth renew). Persona-driven (`personas/instructions/monitor.md`, hot-reload). Main process `monitor_command_server.py` (`args:`, tini-wrapped) runs the tick as a subprocess AND serves a Bearer control plane on `:8769` (status, `pause`/`resume`/`ack`/`force-tick`, read-only Q&A via `monitor_qa` in-pod — the only pod seeing kubectl + forge + `/state`). `deilebot` `/monitor` cog + cluster-question auto-route consume it; `/v1/health` 503 on stale tick → restart.

### 5.6 Observability gotchas — what you see is NOT always reality
1. **`ps`/`pgrep` work** (`procps` in the image). Old image: `cat /proc/*/cmdline | tr '\0' ' '` (`/proc` shows the host PIDNS).
2. **Pipeline logs look empty when idle** — the monitor emits per-tick; short tails get evicted by health probes. Use `--tail=500` / `--since=10m`. Unbuffered (`PYTHONUNBUFFERED=1` + tini), so true silence-with-RUNNING = deadlock (rare) or too-aggressive a tail.
3. **`.lease.json` mtime is NOT liveness for `claude -p`** — the wrapper rewrites `heartbeat_at` every 5 s regardless, lease `pid` = **wrapper** PID. Each dispatch writes `claude_pid`; `_find_active_lease` reports `claude_running` (True iff that PID alive). **Trust `claude_running` (or `pgrep -a claude`), never lease mtime.**
4. **OAuth renew is an operational trap.** The claude-worker access token expires (~hours); the in-pod "refresh" does **not** rotate the refresh token → the whole fleet falls into `401 / WORKER_AUTH_EXPIRED`, looking like a "stuck pipeline" with no code cause. Fix from host: **`python3 infra/k8s/deploy.py k8s claude-renew`** (reads the macOS keychain). Check this FIRST when dispatches mysteriously stall.

### 5.8 The CLI worker fleet (Decisão #51 — pluggable coding CLIs)
The fleet generalizes claude-worker into N pluggable CLI workers. **claude-worker is untouched — one worker among several.** Architecture in `00-VISAO-GERAL.md` Decisão #51; adapter authoring template in `12-PADROES-CODIGO.md` (**CLI Adapter Development**). Operational notes:

- **Single source of truth = the adapter registry.** `infra/k8s/cli_adapters/__init__.py` scans the package and builds `ADAPTERS = {kind: adapter}`; every consumer (`dispatch_resolver` → `VALID_DISPATCHERS`, panel, `deploy.py gen-worker`, NetworkPolicy egress) iterates it. **Add a worker = drop `cli_adapters/<kind>.py`; no consumer edited** (guarded by `test_worker_registry_drives_everything.py`).
- **Success gate, not exit-code.** CLI exit-codes are unreliable: `WorkResult.ok = adapter.parse_output(...).ok AND wrapper_gate()` — the post-run gate (new commit + push, or green tests when the brief demands) is the final authority. `git_strategy`: `brief_driven` (opencode/codex/qwen/goose — the brief drives `git add/commit/push`) vs `cli_autocommit` (aider commits itself, with `--no-attribute-*` so no "(aider)"/`Co-Authored-By`).
- **Native resume per worker (#445, anti-cost-bleed).** Each worker resumes its **native session in the same workdir** instead of re-spending: opencode `run --session <id>`; codex `exec resume <id>` (does NOT accept `--cd`); qwen `--resume <id>`; goose `run --name <id> --resume` (deterministic named session — id = task_id); aider `--restore-chat-history` (workdir-keyed). `supports_resume=True` → the manifest gen provisions a **per-worker PVC `<kind>-worker-home`** + a cleanup CronJob. A provider error (402/429/insufficient) is classified **INCOMPLETE** (`_worker_core.classify_provider_error`) so the pipeline resumes rather than declaring "complete".
- **Durable cost + audit.** `cli_worker_server` harvests each session's cost into a durable ledger (`<root>/.cost-ledger.jsonl`, **dedup by task_id**, env `DEILE_CLI_WORKER_COST_LEDGER_PATH`) **BEFORE pruning** the bulky `.progress/*.stdout.log` (mirrors claude #445). `infra/k8s/jsonl_cost.py` (`FLEET_PRICING_BY_SUBSTRING` + aggregation) is the single pricing source; `infra/k8s/fleet_progress_parse.py` is the single source of the per-kind `.progress` parsers; `infra/k8s/fleet_tokens_audit.py` (panel **`[T]`okens** screen) reads ledger (pruned) + live `.progress`, dedup by task_id. **Never prune without harvesting** (the cost-ledger incident: 337 transcripts deleted).
- **Auth — two modes.** `env` (API key, doesn't expire — preferred for automation, default) and `oauth_file` (mounted OAuth cred, opt-in via `DEILE_<KIND>_AUTH=oauth`). codex prefers OpenAI direct (`wire_api=responses` rules out most of OpenRouter); qwen via the `OPENAI_*` triad (Dashscope/OpenRouter — §5.4 "Configurar provider…" recipe); aider/goose via OpenRouter. Keys live in Secret `cli-worker-keys`; the worker bearer in Secret file `/run/secrets/cli-worker/CLI_WORKER_BEARER_TOKEN`. OAuth-capable adapters install via `cli-worker-login` (§5.4).
- **goose model gotcha.** The goose catalog `id` is **provider-prefixed** (`openrouter/deepseek/...`): `build_argv` splits on the **first `/`** → `--provider`/`--model`. Without the prefix goose reads the 1st segment as the provider and fails "Unknown provider" (there is no `GOOSE_PROVIDER` in the Deployment). `--max-turns` is env-overridable via `DEILE_GOOSE_MAX_TURNS`. (`cli_adapters/goose.py`.)
- **Image.** Single `Dockerfile.cli-worker`, multi-stage `deps`→`base`→`final`; `WORKER_KIND` build-arg only in `final` (buildkit caches deps/base once, containerd dedups by digest). Build via `k8s build-cli-workers [--kind <k>]`; generated manifests in `manifests/generated/` are gitignored/ephemeral.
- **Known gaps (open issues — follow-up, NOT done):** **#638** push-central cost via `UsageRepository` (today: pull-via-exec + per-PVC ledger); **#639** repo allowlist NOT enforced at dispatch (exfil risk — production pre-req); **#640** 3 pre-existing red tests in `orchestration/pipeline` (dirty baseline).

| **Manifest `env:` blocks** | per-pod hardcoded: ports, paths, autostart flags, whitelists. | PR in repo. |

**Macro env categories** (full list in `.env.example`): LLM keys (`ANTHROPIC_/OPENAI_/DEEPSEEK_/GOOGLE_API_KEY`, ≥1 required) · forges (`GITHUB_TOKEN`, `GITLAB_TOKEN`/`GL_TOKEN`, `DEILE_FORGE_{KIND,REPO}`, `DEILE_*_HOST`) · bot (`DEILE_BOT_*`) · workers (`DEILE_WORKER_*`, `DEILE_CLAUDE_WORKER_*`; `DEILE_WORKER_TASK_TIMEOUT_S=7200`) · CLI fleet (`DEILE_CLI_WORKER_{PORT,HOST,ROOT,HOME,TASK_TIMEOUT_S,CLEANUP_INTERVAL_S,CLEANUP_RETENTION_DAYS,PROGRESS_RETENTION_DAYS,PROGRESS_GRACE_S,COST_LEDGER_PATH}`, `DEILE_<KIND>_AUTH` (oauth opt-in), `DEILE_CLI_<KIND>_ENV_<VAR>` (per-worker env injection, §5.4), `DEILE_GOOSE_MAX_TURNS`, `OPENROUTER_API_KEY`) · per-stage `DEILE_PIPELINE_{DISPATCH,MODEL,REASONING}_<STAGE>` · subagents (`DEILE_SUBAGENT_*`) · loop guard (`DEILE_MAX_TOOL_ITERATIONS`=100) · cron (`DEILE_CRON_*`) · OTLP (`DEILE_OTLP_*`) · status (`DEILE_PIPELINE_STATUS_*`) · `DEILE_K8S_{NAMESPACE,DEPLOY_PROFILE}`, `DEILE_RUNTIME_DIR`.

**Bootstrap:** `k8s up` auto-generates missing bearer tokens (`secrets.token_urlsafe(32)`) and **persists them back to `.env`** (#354). Discord token only for `full`. Remaining gap = the **spread itself** (no unified "where is config X" viewer). ## 7. Forge — GitHub & GitLab

DEILE is **forge-agnostic** (#297): same pipeline/briefs/tools over GitHub (cloud + GHES) and GitLab (cloud + self-hosted). `orchestration/forge/` hides it behind `ForgeClient` (`GitHubForge` via `gh`, `GitLabForge` via `glab` + REST v4); the pipeline never calls CLIs directly (`pipeline/github_client.py` = deprecation shim). Full setup + vocab: **README §Forge**.

| Env var | Default | Note |
|---|---|---|
| `DEILE_FORGE_KIND` | `auto` | `github`\|`gitlab`; `auto` detects by host then path heuristic. |
| `DEILE_FORGE_REPO` | — | canonical repo target. **`DEILE_PIPELINE_REPO` was removed** from domain code (only survives in manifests as the init clone target); `settings.json pipeline.repo` is the fallback. |
| `DEILE_GITHUB_HOST` / `DEILE_GITLAB_HOST` | `github.com` / `gitlab.com` | GHES (CSV ok) / self-hosted. |
| `GITHUB_TOKEN` (`repo`+`workflow`, **no `read:org`**) · `GITLAB_TOKEN`/`GL_TOKEN` (`api`,`read_repository`,`write_repository`) | — | ≥1 token required. |

Vocabulary the code aligns: **PR↔MR** · comment↔note · `requested_reviewers[]`↔`reviewers[]` · `pr.number`↔`mr.iid` · `/pull/N`↔`/-/merge_requests/N` · `.github/ISSUE_TEMPLATE/`↔`.gitlab/issue_templates/` · `gh`↔`glab`. Detection layered: `DEILE_FORGE_KIND` > URL host / `DEILE_*_HOST` > path heuristic (3+ segments → GitLab, 2 → GitHub). `secrets_scanner` catches GitHub (`ghp_/gho_/ghu_/ghs_/ghr_/github_pat_`) **and** GitLab (`glpat-/gldt-/glptt-/glsoat-`) tokens.

Multi-forge in one **CLI** session via the `ForgeRouter` singleton (both tokens set). The **pipeline** is per-repo/per-forge — run two instances for GH+GL in parallel (`MonitorIdentity` + shard, Decisão #18).

## 8. Task playbooks (recipes for working autonomously)

- **Ship a code change to the cluster** → edit → `deploy.py k8s build --restart --yes` (image is baked; **no rebuild = no effect**) → confirm with `k8s status` + `k8s logs`. Config-only change to the pipeline pod → `kubectl set env` + `k8s restart` avoids a rebuild.
- **"Pipeline looks stuck"** → (1) **OAuth first** — `deploy.py k8s claude-renew` from the host (expired token masquerades as stuck, §5.6); (2) liveness via `claude_running` / `pgrep -a claude`, never lease mtime; (3) `k8s logs deile-pipeline --tail=500`; (4) panel `[t]` tokens or status server `/v1/pipeline-status`. `/babysit-deile` arms watchers for exactly this.
- **Work on a brief / the quality gate** → briefs in `briefs.py`; remember implement=subset, PR=full suite, no cov gate; after editing run `pytest deile/tests/orchestration/pipeline/test_briefs* -v` (invariants test-guarded).
- **Tune per-stage routing/model/reasoning** → panel `[d]` matrix, or env `DEILE_PIPELINE_{DISPATCH,MODEL,REASONING}_<STAGE>` (rollout `deile-pipeline`, no rebuild).
- **Add a tool/command/parser/persona/skill/provider** → §3.5 map + templates in `12-PADROES-CODIGO.md`; then full suite (mind ordering pollution, §4).
- **Add a CLI worker to the fleet** → write **one** adapter `infra/k8s/cli_adapters/<kind>.py` (CLI Adapter Development template in `12-PADROES-CODIGO.md`) + tests; **no consumer is edited** (`test_worker_registry_drives_everything.py` proves it). Then `k8s build-cli-workers --kind <k>` → `k8s cli-worker-install <kind>` (or `cli-worker-login` if OAuth) → panel `[d]` to route a stage to it. Details §5.8.
- **Commit & integrate** → commit **per phase**; **full green suite + self-review before every commit**; **review the staged diff against `README.md`/touched pillars and rewrite stale sections in the same commit** — code and docs must never disagree; English title / PT-BR body; no `Co-Authored-By`. Finished branch → `superpowers:finishing-a-development-branch`.
- **Restart the bot** → kill old instances/pods first; two live bots double-process events.
- **Visual panel change** (`_panel*.py`) → **offer the Human a screenshot** (SVG via `Console.save_svg`) — don't auto-generate.

## 9. Hard-won operational lessons (don't re-pay the cost)

- **claude-worker resume context measurement bug (fixed `065e3ea`):** the gate summed `cache_read` across all rounds (11.5M in a ~70K session) vs a fixed 100K ceiling → every long review restarted. Fix = peak-of-1-round + 80% of the real window. `claude -p` has **no tunable auto-compact** (only via SDK). Long sessions restarting → suspect context-measurement, not the model.
- **`ultracode` on claude-worker** needs `claude ≥ 2.1.158` (pinned in the Dockerfile) and injects the keyword "workflow" + `xhigh`. Bumping the pin can hit a **Docker layer-cache** trap — force a clean build layer.
- **`infra/k8s/` packaging + destructive-code fail-safe:** the module ships in the image via explicit `COPY` + a `.dockerignore` exception. **Never prune without harvesting first**, and always test the `None` path — the cost-ledger incident (337 transcripts deleted) came from pruning before harvesting.
- **Verify before asserting:** never claim "works / it's live" without empirical proof; say what you couldn't test. (`feedback_honesty_verify`.)
- **`/proc`-first skepticism** for claude-worker liveness; run a skeptical delta-loop, not a hopeful one (`/babysit-deile` encodes this).
- More incident detail in persistent memory (`MEMORY.md`) — cost accounting, loop-guards history, refinement-gate/state-machine deep notes.

## 10. Gotchas (not in the system design)

- **At least one provider key is required at startup** (`ANTHROPIC_/OPENAI_/DEEPSEEK_/GOOGLE_API_KEY`); `bootstrap_providers()` (`core/models/bootstrap.py`) does conditional registration.
- **`deilebot` is a separate repo** (`elimarcavalli/deilebot`). `pip install -e ".[bot]"` (git-URL extra) for cloud/CI; clone it + `pip install -e ./deilebot` to hack on the bot. `messaging.discord_*` tools auto-register **only** when `import deilebot` succeeds **and** `DEILE_BOT_ENDPOINT` + `DEILE_BOT_AUTH_TOKEN` are set. Run the bot from the `deile/` repo root.
- **Touching messaging tools?** Open `08-SEGURANCA.md` first — DM and role-mention tools are `SecurityLevel.DANGEROUS`, gated by `ApprovalSystem` (`orchestration/approval_system.py`); that gate is non-trivial.
- **`Settings` is a singleton** — `from deile.config.settings import get_settings`, never `Settings()`.
- **Personas are MD-driven** — edit `personas/instructions/*.md`; no Python change.
- **`CLAUDE.md` is versioned** — nothing in `.gitignore` ignores it (lists `.claude/`, `.openclaude/*`, `.claude-worktrees/` — **no** `*claude*` glob nor `!CLAUDE.md` negation; don't go hunting for one).
- **Discord interactions** → use native modals/buttons/selects; check the docs before implementing.
- **The tool registry does NOT enforce permissions/audit.** "Permission-before-action + typed audit on every tool" is **not** in the central exec path (`tool_loop_executor`) — only `bash_tool` checks `PermissionManager` (and allows-all when no manager injected); other tools run ungated (Decisão #29 deferred). The reliably-firing audit event is the **loop-guard abort** (`SUSPICIOUS_ACTIVITY`). `secrets_scanner` redaction IS wired in the session store (`core/session_store.py`), covering GH/GL/AWS/bearer/private-key patterns. Rely on loop guard + NetworkPolicy + tool whitelist for containment, not a per-tool gate that mostly isn't there.

## 11. Running DEILE for empirical testing

You may invoke `python3 deile.py` (or call the agent programmatically) to test against the real LLM (modest token spend approved). **Where files go (don't conflate):** `test-your-might/<nick>/` = **DEILE writes** (its own test artifacts, keeps root clean); `deile/tests/might/<nick>/` = **you write** real-LLM scripts (cost tokens, isolated from pytest); `deile/tests/` (rest) = **you write** regular pytest (no API calls).

- Budget proportional (smoke test = 1–4 messages); reuse the **same process** across multi-turn probes (history continuity).
- Bootstrap by mirroring `deile.py`: `ConfigManager().load_config()` + `bootstrap_providers(router=get_model_router())` (`bootstrap_providers()` alone registers 0 providers — the router is the singleton the agent reads).
- **Capture output, strip ANSI, report verbatim** what DEILE said + tools called; never paraphrase. If DEILE asks a confirmation you can't answer, kill it and report that.
- **Opening an issue** → read `.github/ISSUE_TEMPLATE/*.md`, pick the right one (default `intent`).

## 12. SQL / database operations

All SQL scripts are the Human's to run. On a DB error during a task, **stop and tell the operator which script to execute** — never run migrations or schema changes yourself. DEILE's own stores are SQLite, auto-created at runtime (no versioned SQL): tasks `./.deile/db/tasks.db`, usage `~/.deile/db/usage.db`, episodic `episodes.db`, cron `data/cron.db`; the claude-worker cost ledger is JSONL at `~/.claude/cost-ledger.jsonl`, and each CLI fleet worker keeps its own JSONL ledger at `<root>/.cost-ledger.jsonl` on its PVC (§5.8).
